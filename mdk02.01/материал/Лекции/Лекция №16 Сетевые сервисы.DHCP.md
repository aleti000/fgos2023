## Введение

### Понятие DHCP и его роль в сетевой инфраструктуре

Dynamic Host Configuration Protocol (DHCP) — это сетевой протокол, предназначенный для автоматической настройки параметров сетевого подключения клиентских устройств в IP-сетях. DHCP играет ключевую роль в современных сетевых инфраструктурах, обеспечивая централизованное управление сетевыми параметрами и упрощая администрирование сетей.

**Основные функции DHCP:**
- Автоматическое назначение IP-адресов
- Настройка маски подсети
- Указание шлюза по умолчанию
- Назначение DNS-серверов
- Настройка других сетевых параметров

**Преимущества использования DHCP:**
- Снижение нагрузки на сетевых администраторов
- Исключение ошибок при ручной настройке
- Эффективное использование IP-адресного пространства
- Централизованное управление сетевыми параметрами
- Возможность быстрого изменения конфигурации сети

### История развития DHCP

**Предшественники DHCP:**
- **BOOTP (Bootstrap Protocol)** — протокол, разработанный в 1985 году для загрузки дисковыхless-систем
- BOOTP предоставлял только базовую информацию: IP-адрес, имя хоста, адрес загрузочного сервера

**Развитие DHCP:**
- **1993 год** — RFC 1531 определил DHCP как расширение BOOTP
- **1997 год** — RFC 2131 и RFC 2132 стали стандартами DHCP
- **2003 год** — RFC 3315 определил DHCP для IPv6 (DHCPv6)

**Современное состояние:**
- DHCP стал стандартом де-факто для автоматической настройки сетей
- Поддерживается всеми современными операционными системами
- Интегрирован в большинство сетевого оборудования

### Преимущества использования DHCP перед статической настройкой

**Для администраторов:**
- Автоматизация рутинных задач
- Снижение вероятности конфликтов IP-адресов
- Возможность централизованного управления
- Упрощение масштабирования сети

**Для пользователей:**
- Автоматическая настройка без вмешательства пользователя
- Возможность подключения к сети "из коробки"
- Стабильная работа в гостевых сетях

**Для сети в целом:**
- Эффективное использование IP-адресного пространства
- Гибкость в управлении сетевыми ресурсами
- Возможность быстрого реагирования на изменения

### Принципы работы DHCP

#### Архитектура клиент-серверной модели

**Компоненты DHCP-инфраструктуры:**

1. **DHCP-сервер** — устройство, отвечающее за распределение IP-адресов и других сетевых параметров
2. **DHCP-клиент** — устройство, запрашивающее и использующее сетевые параметры
3. **DHCP-релей** — устройство для передачи DHCP-запросов между сегментами сети

**Типы DHCP-серверов:**
- **Выделенные DHCP-серверы** — специализированные устройства или серверы
- **Встроенные DHCP-серверы** — функция, реализованная в маршрутизаторах, коммутаторах
- **Программные DHCP-серверы** — ПО, установленное на универсальных серверах

#### Процесс получения IP-адреса (DORA)

**Фаза DORA (Discover, Offer, Request, Acknowledge):**

**1. Discover (Поиск)**
```bash
# Клиент рассылает широковещательный запрос
DHCP Discover → (Broadcast)
```
- Клиент отправляет широковещательный запрос DHCP Discover
- Запрос содержит MAC-адрес клиента
- Цель: найти доступные DHCP-серверы в сети

**2. Offer (Предложение)**
```bash
# Сервер отвечает предложением конфигурации
DHCP Offer → (Unicast/Broadcast)
```
- DHCP-сервер отвечает на запрос
- В ответе содержится предложенный IP-адрес и параметры
- Сервер резервирует адрес для данного клиента

**3. Request (Запрос)**
```bash
# Клиент запрашивает предложенный адрес
DHCP Request → (Broadcast)
```
- Клиент выбирает одно из предложений
- Отправляет запрос на получение указанного адреса
- Запрос может быть широковещательным для уведомления других серверов

**4. Acknowledge (Подтверждение)**
```bash
# Сервер подтверждает выделение адреса
DHCP Ack → (Unicast/Broadcast)
```
- Сервер подтверждает выделение IP-адреса
- Передает все необходимые сетевые параметры
- Клиент начинает использовать полученные параметры

#### Временные параметры аренды IP-адресов

**Понятие аренды:**
- IP-адрес выделяется клиенту на определенный период времени
- По истечении срока аренды адрес может быть перераспределен
- Клиент может продлевать аренду автоматически

**Типы аренды:**
- **Временная аренда** — стандартный режим работы
- **Бесконечная аренда** — адрес закрепляется за клиентом навсегда
- **Резервирование** — адрес зарезервирован за конкретным MAC-адресом

**Процесс продления аренды:**
```bash
# Автоматическое продление аренды
T1 (50% срока) → Renewal Request
T2 (87.5% срока) → Rebind Request
T3 (100% срока) → Release/Reboot
```

#### DHCP-опции и их назначение

**Структура DHCP-опций:**
- Каждая опция имеет уникальный номер
- Опции содержат дополнительные параметры конфигурации
- Опции могут быть обязательными или опциональными

**Основные DHCP-опции:**

| Опция | Номер | Назначение |
|-------|-------|------------|
| Subnet Mask | 1 | Маска подсети |
| Router | 3 | Шлюз по умолчанию |
| DNS Servers | 6 | DNS-серверы |
| Domain Name | 15 | Доменное имя |
| Broadcast Address | 28 | Адрес широковещательной рассылки |
| NTP Servers | 42 | NTP-серверы |
| WINS Servers | 44 | WINS-серверы |
| Lease Time | 51 | Время аренды |
| Message Type | 53 | Тип DHCP-сообщения |
| Server Identifier | 54 | Идентификатор сервера |
| Parameter Request List | 55 | Список запрашиваемых параметров |
| Classless Static Routes | 121 | Маршруты без классов |

**Примеры использования опций:**
```bash
# Настройка DNS-серверов
option domain-name-servers 8.8.8.8, 1.1.1.1;

# Настройка времени аренды
default-lease-time 86400;
max-lease-time 172800;

# Настройка маршрутов
option classless-static-routes 10.0.0.0/8 192.168.1.1;
```

### DHCP-серверы и их типы

#### Основные функции DHCP-сервера

**1. Управление IP-адресным пространством:**
- Определение диапазонов IP-адресов (пулов)
- Контроль за использованием адресов
- Предотвращение конфликтов IP-адресов

**2. Распределение сетевых параметров:**
- Назначение IP-адресов клиентам
- Настройка маски подсети
- Указание шлюза по умолчанию
- Назначение DNS-серверов

**3. Управление арендой:**
- Контроль сроков аренды
- Автоматическое продление аренды
- Освобождение неиспользуемых адресов

**4. Ведение учета:**
- Логирование выданных адресов
- Отслеживание состояния аренды
- Ведение статистики использования

#### Различные типы DHCP-серверов

**1. Программные DHCP-серверы:**
- **ISC DHCP Server** — наиболее популярный开源-сервер
- **Kea DHCP** — современная альтернатива от ISC
- **Windows DHCP Server** — встроенный сервер Windows Server

**2. Аппаратные DHCP-серверы:**
- **Маршрутизаторы** — встроенные DHCP-серверы
- **Коммутаторы** — серверы в управляемых коммутаторах
- **Точки доступа Wi-Fi** — встроенные серверы

**3. Виртуальные DHCP-серверы:**
- **VMware vSphere** — DHCP-серверы в виртуальной среде
- **Cloud DHCP** — сервисы в облачных платформах

#### Резервирование IP-адресов

**Типы резервирования:**

**1. Статическое резервирование:**
```bash
# Резервирование по MAC-адресу
host printer {
    hardware ethernet 00:11:22:33:44:55;
    fixed-address 192.168.1.100;
}
```

**2. Динамическое резервирование:**
- Адрес резервируется на время аренды
- После освобождения может быть выдан другому клиенту

**3. Перманентное резервирование:**
- Адрес закрепляется за устройством навсегда
- Используется для серверов и сетевого оборудования

**Преимущества резервирования:**
- Гарантированное получение одного и того же адреса
- Упрощение настройки статических маршрутов
- Обеспечение стабильной работы сервисов

#### Работа с несколькими подсетями

**1. DHCP-релей (DHCP Relay):**
```bash
# Настройка DHCP-релея на маршрутизаторе
interface GigabitEthernet0/0
 ip helper-address 192.168.1.100
```

**2. Многосайтовая конфигурация:**
```bash
# Конфигурация для нескольких подсетей
subnet 192.168.1.0 netmask 255.255.255.0 {
    range 192.168.1.10 192.168.1.100;
    option routers 192.168.1.1;
}

subnet 192.168.2.0 netmask 255.255.255.0 {
    range 192.168.2.10 192.168.2.100;
    option routers 192.168.2.1;
}
```

**3. SuperScope (Надобласть):**
- Объединение нескольких подсетей в одну логическую область
- Используется для сложных сетевых топологий
- Позволяет эффективно использовать IP-адресное пространство

### Безопасность DHCP

#### DHCP-спуфинг и методы защиты

**1. DHCP-спуфинг (DHCP Spoofing):**
- Злоумышленник развертывает ложный DHCP-сервер
- Клиенты получают неправильные сетевые параметры
- Возможна перенаправление трафика через злоумышленника

**2. Методы защиты:**

**DHCP Snooping:**
```bash
# Настройка DHCP Snooping на коммутаторе
ip dhcp snooping
ip dhcp snooping vlan 10
interface GigabitEthernet0/1
 ip dhcp snooping trust
```

**Port Security:**
```bash
# Ограничение MAC-адресов на порту
switchport port-security
switchport port-security maximum 1
switchport port-security violation restrict
```

**Dynamic ARP Inspection (DAI):**
```bash
# Проверка ARP-таблиц
ip arp inspection vlan 10
ip arp inspection validate src-mac dst-mac ip
```

#### DHCP snooping

**Принцип работы DHCP Snooping:**
- Коммутатор анализирует DHCP-пакеты
- Создает таблицу привязки MAC-адресов к портам
- Блокирует DHCP-пакеты с непроверенных портов

**Конфигурация DHCP Snooping:**
```bash
# Включение DHCP Snooping
ip dhcp snooping

# Настройка VLAN
ip dhcp snooping vlan 10,20

# Настройка доверенных портов
interface GigabitEthernet0/1
 ip dhcp snooping trust

# Настройка максимального количества DHCP-пакетов
ip dhcp snooping limit rate 100
```

**Преимущества DHCP Snooping:**
- Защита от DHCP-спуфинга
- Контроль за DHCP-трафиком
- Создание базы данных для других функций безопасности

#### Взаимодействие с другими сетевыми службами

**1. Интеграция с DNS:**
- DHCP может обновлять DNS-записи автоматически
- Динамическое обновление зон DNS
- Синхронизация информации о хостах

**2. Интеграция с Active Directory:**
- Аутентификация DHCP-серверов
- Централизованное управление
- Интеграция с групповыми политиками

**3. Взаимодействие с сетевыми устройствами:**
- Синхронизация с маршрутизаторами
- Интеграция с системами мониторинга
- Обмен информацией о состоянии сети

### Конфигурирование DHCP-сервера

#### Структура конфигурационного файла /etc/dhcp/dhcpd.conf

**1. Основные секции конфигурации:**

```bash
# Просмотр структуры конфигурационного файла
cat /etc/dhcp/dhcpd.conf
```

**Типичная структура файла:**
```conf
# Глобальные параметры
option domain-name "example.com";
option domain-name-servers 8.8.8.8, 1.1.1.1;
default-lease-time 86400;
max-lease-time 172800;

# Настройка интерфейса
authoritative;

# Объявление подсети
subnet 192.168.1.0 netmask 255.255.255.0 {
    range 192.168.1.10 192.168.1.100;
    option routers 192.168.1.1;
    option subnet-mask 255.255.255.0;
}
```

#### Основные директивы и параметры

**1. Глобальные параметры:**

```conf
# Время аренды по умолчанию (в секундах)
default-lease-time 86400;

# Максимальное время аренды (в секундах)
max-lease-time 172800;

# Авторитетный сервер (отвечает на все запросы)
authoritative;

# Логирование
log-facility local7;
```

**2. Сетевые параметры:**

```conf
# Доменное имя
option domain-name "example.com";

# DNS-серверы
option domain-name-servers 8.8.8.8, 1.1.1.1;

# NTP-серверы
option ntp-servers 0.pool.ntp.org;

# WINS-серверы (для Windows-сетей)
option netbios-name-servers 192.168.1.10;
option netbios-node-type 8;
```

**3. Параметры маршрутизации:**

```conf
# Шлюз по умолчанию
option routers 192.168.1.1;

# Маршруты без классов
option classless-static-routes 10.0.0.0/8 192.168.1.1;
```

#### Настройка пулов адресов

**1. Базовая конфигурация пула:**

```conf
# Объявление подсети
subnet 192.168.1.0 netmask 255.255.255.0 {
    # Пул адресов для выдачи
    range 192.168.1.10 192.168.1.100;
    
    # Статический пул для серверов
    range 192.168.1.101 192.168.1.110;
}
```

**2. Множественные пулы в одной подсети:**

```conf
subnet 192.168.1.0 netmask 255.255.255.0 {
    # Пул для пользовательских устройств
    range 192.168.1.10 192.168.1.50;
    
    # Пул для гостевых устройств
    range 192.168.1.51 192.168.1.100;
    
    # Пул для IoT-устройств
    range 192.168.1.101 192.168.1.150;
}
```

**3. Пулы с разными параметрами:**

```conf
# Пул для офисной сети
subnet 192.168.1.0 netmask 255.255.255.0 {
    range 192.168.1.10 192.168.1.50;
    option routers 192.168.1.1;
    option domain-name-servers 192.168.1.10;
}

# Пул для гостевой сети
subnet 192.168.2.0 netmask 255.255.255.0 {
    range 192.168.2.10 192.168.2.50;
    option routers 192.168.2.1;
    option domain-name-servers 8.8.8.8;
}
```

#### Конфигурирование опций

**1. Глобальные опции:**

```conf
# Глобальные настройки DNS
option domain-name "company.local";
option domain-name-servers 192.168.1.10, 192.168.1.11;

# Глобальные настройки времени
option ntp-servers 0.pool.ntp.org, 1.pool.ntp.org;

# Глобальные настройки маршрутизации
option routers 192.168.1.1;
```

**2. Опции для конкретной подсети:**

```conf
subnet 192.168.1.0 netmask 255.255.255.0 {
    range 192.168.1.10 192.168.1.100;
    
    # Локальные DNS-серверы
    option domain-name-servers 192.168.1.10;
    
    # Внутренний NTP-сервер
    option ntp-servers 192.168.1.20;
    
    # Внутренний WINS-сервер
    option netbios-name-servers 192.168.1.30;
}
```

**3. Опции для конкретных хостов:**

```conf
# Конфигурация для принтера
host printer {
    hardware ethernet 00:11:22:33:44:55;
    fixed-address 192.168.1.100;
    option host-name "printer-01";
    option domain-name-servers 8.8.8.8;
}

# Конфигурация для сервера
host server {
    hardware ethernet 00:AA:BB:CC:DD:EE;
    fixed-address 192.168.1.200;
    option host-name "web-server";
    option domain-name-servers 192.168.1.10;
}
```

## Глоссарий терминов

- **DHCP (Dynamic Host Configuration Protocol)** — протокол динамической настройки хоста
- **DORA** — последовательность сообщений DHCP (Discover, Offer, Request, Acknowledge)
- **Аренда IP-адреса** — временный период использования IP-адреса клиентом
- **DHCP-сервер** — устройство, предоставляющее IP-адреса и сетевые параметры
- **DHCP-клиент** — устройство, получающее IP-адрес и параметры от DHCP-сервера
- **DHCP-релей** — устройство для передачи DHCP-запросов между сегментами сети
- **Пул адресов** — диапазон IP-адресов, доступных для выдачи клиентам
- **Резервирование** — закрепление IP-адреса за конкретным MAC-адресом
- **DHCP-спуфинг** — атака, при которой злоумышленник развертывает ложный DHCP-сервер
- **DHCP snooping** — технология защиты от DHCP-спуфинга
- **Авторитетный сервер** — DHCP-сервер, отвечающий на все запросы в сети
- **Время аренды** — период, на который выделяется IP-адрес клиенту
- **DHCP-опции** — дополнительные параметры конфигурации, передаваемые клиенту
- **Файл аренд** — файл, содержащий информацию об активных и завершенных арендах
- **Широковещательный запрос** — сетевой запрос, отправляемый всем устройствам в сети
