## 1. Подготовка к добавлению узлов в домен

### 1.1. Требования к клиентским машинам

**Общие требования:**
- Сетевая связность с контроллером домена
- Корректно настроенный DNS (указание на контроллер домена)
- Синхронизация времени (разница не более 5 минут)
- Поддерживаемая операционная система

**Требования для Linux-клиентов:**
- Установленные пакеты: realmd, sssd, samba-common-tools, krb5-user
- Настроенный сетевой интерфейс
- Права администратора для выполнения операций

**Требования для Windows-клиентов:**
- Поддерживаемые версии: Windows 10/11, Windows Server 2016+
- Сетевой профиль: частная или доменная сеть
- Включенная служба "Клиент DNS"

### 1.2. Предварительная настройка сети и DNS

**Настройка DNS на клиенте:**
```bash
# Для Linux (временная настройка)
echo "nameserver 192.168.1.1" > /etc/resolv.conf

# Для Linux (постоянная настройка в ALT Linux)
nmcli connection modify "eth0" ipv4.dns "192.168.1.1"
nmcli connection up "eth0"

# Для Windows (через PowerShell)
Set-DnsClientServerAddress -InterfaceIndex 1 -ServerAddresses 192.168.1.1
```

**Проверка разрешения имен:**
```bash
# Проверка разрешения имени контроллера домена
nslookup dc1.company.local

# Проверка SRV записей
nslookup -type=srv _ldap._tcp.company.local

# Проверка обратного разрешения
nslookup 192.168.1.1
```

**Синхронизация времени:**
```bash
# Для Linux (установка NTP)
apt-get install ntp
systemctl enable --now ntp

# Проверка синхронизации
ntpq -p

# Для Windows (настройка времени)
w32tm /config /manualpeerlist:"dc1.company.local" /syncfromflags:manual /reliable:yes
w32tm /resync
```

## 2. Добавление Linux-клиента в домен Samba AD

### 2.1. Установка необходимых пакетов

**Для ALT Linux:**
```bash
# Обновление системы
apt-get update

# Установка пакетов для работы с доменом
apt-get install realmd sssd sssd-tools adcli samba-common-tools krb5-user oddjob oddjob-mkhomedir

# Проверка установленных пакетов
rpm -qa | grep -E "(realmd|sssd|adcli|krb5)"
```

**Назначение пакетов:**
- **realmd** — утилита для присоединения к домену
- **sssd** — служба кэширования учетных данных и аутентификации
- **adcli** — утилиты для работы с Active Directory
- **samba-common-tools** — инструменты Samba (kinit, klist)
- **krb5-user** — клиент Kerberos
- **oddjob** — служба для автоматического создания домашних каталогов

### 2.2. Настройка DNS клиента

**Проверка текущего DNS:**
```bash
# Просмотр текущих DNS серверов
cat /etc/resolv.conf

# Проверка через nmcli
nmcli device show eth0 | grep IP4.DNS
```

**Настройка DNS на контроллер домена:**
```bash
# Временная настройка
echo "nameserver 192.168.1.1" > /etc/resolv.conf

# Постоянная настройка через nmcli
nmcli connection modify "eth0" ipv4.ignore-auto-dns yes
nmcli connection modify "eth0" ipv4.dns "192.168.1.1"
nmcli connection up "eth0"

# Проверка
nslookup company.local
```

### 2.3. Присоединение к домену через realm

**Проверка возможности присоединения:**
```bash
# Проверка доступности домена
realm discover company.local

# Пример вывода:
# company.local
#   type: kerberos
#   realm-name: COMPANY.LOCAL
#   domain-name: company.local
#   configured: no
#   server-software: samba
#   client-software: sssd
#   required-package: sssd-tools
#   required-package: sssd
#   required-package: libnss-sss*
#   required-package: libpam-sss*
#   required-package: adcli
```

**Присоединение к домену:**
```bash
# Присоединение с указанием пользователя администратора
realm join company.local -U Administrator

# Система запросит пароль администратора домена

# Альтернативно, можно указать пароль в команде (не рекомендуется из-за безопасности)
echo "password" | realm join company.local -U Administrator --stdin

# Проверка членства в домене
realm list

# Пример вывода:
# company.local
#   type: kerberos
#   realm-name: COMPANY.LOCAL
#   domain-name: company.local
#   configured: kerberos-member
#   server-software: samba
#   client-software: sssd
```

### 2.4. Настройка SSSD (System Security Services Daemon)

**Конфигурационный файл SSSD:**
```bash
# Файл конфигурации: /etc/sssd/sssd.conf

[sssd]
domains = company.local
config_file_version = 2
services = nss, pam

[domain/company.local]
id_provider = ad
access_provider = ad
chpass_provider = ad
ldap_id_mapping = True
fallback_homedir = /home/%u@%d
override_homedir = /home/%u
ldap_schema = rfc2307
enumerate = False
cache_credentials = True
use_fully_qualified_names = False
ldap_tls_cacertdir = /etc/pki/tls/certs
ldap_tls_cacert = /etc/pki/tls/certs/ca-bundle.crt
krb5_realm = COMPANY.LOCAL
ldap_kerberos_realm = COMPANY.LOCAL
```

**Автоматическое создание конфигурации:**
```bash
# SSSD автоматически создает конфигурацию при присоединении к домену
# Проверка созданного файла
cat /etc/sssd/sssd.conf

# Установка правильных прав (критично важно!)
chmod 600 /etc/sssd/sssd.conf
chown root:root /etc/sssd/sssd.conf
```

**Настройка автоматического создания домашних каталогов:**
```bash
# Включение службы oddjobd
systemctl enable --now oddjobd

# Настройка PAM для автоматического создания домашних каталогов
authconfig --enablemkhomedir --update

# Или вручную создать конфигурацию PAM
cat >> /etc/pam.d/system-auth << EOF
session required pam_mkhomedir.so skel=/etc/skel/ umask=0022
EOF
```

**Перезапуск служб:**
```bash
# Перезапуск SSSD
systemctl restart sssd

# Проверка статуса
systemctl status sssd

# Просмотр логов SSSD
journalctl -u sssd -f
```

### 2.5. Проверка работоспособности

**Проверка доменных пользователей:**
```bash
# Получение информации о доменном пользователе

# Пример вывода:
# uid=10001(Administrator) gid=10000(domain users) groups=10000(domain users),10001(domain admins)

# Поиск пользователя в домене
getent passwd Administrator

# Поиск группы в домене

```

**Тестирование аутентификации:**
```bash

# Проверка тикета

#

su - Administrator
```

**Проверка входа в систему:**
```bash

login Administrator

# Или через SSH (если разрешено)
ssh Administrator@client.company.local
```

## 3. Добавление Windows-клиента в домен

### 3.1. Настройка сетевого подключения

**Настройка статического IP (опционально):**
```powershell
# Через PowerShell (от администратора)
New-NetIPAddress -InterfaceAlias "Ethernet" -IPAddress 192.168.1.100 -PrefixLength 24 -DefaultGateway 192.168.1.1

# Настройка DNS
Set-DnsClientServerAddress -InterfaceIndex 1 -ServerAddresses 192.168.1.1
```

**Проверка сетевого подключения:**
```powershell
# Проверка связи с контроллером домена
ping dc1.company.local

# Проверка разрешения имен
nslookup company.local

# Проверка доступности LDAP
Test-NetConnection -ComputerName dc1.company.local -Port 389
```

### 3.2. Настройка DNS

**Через графический интерфейс:**
1. Панель управления → Сеть и Интернет → Сетевые подключения
2. Свойства подключения → IP версии 4 (TCP/IPv4) → Свойства
3. Использовать следующие адреса DNS-серверов: 192.168.1.1
4. Дополнительно → DNS → Добавить: company.local

**Через PowerShell:**
```powershell
# Настройка DNS сервера
Set-DnsClientServerAddress -InterfaceIndex 1 -ServerAddresses 192.168.1.1

# Настройка DNS суффикса
Set-DnsClient -InterfaceIndex 1 -ConnectionSpecificSuffix "company.local"

# Проверка
ipconfig /all
```

### 3.3. Присоединение к домену

**Через графический интерфейс:**
1. Система → Дополнительные параметры системы
2. Имя компьютера → Изменить
3. Является членом домена: company.local
4. Ввести учетные данные администратора домена
5. Перезагрузить компьютер

**Через PowerShell:**
```powershell
# Присоединение к домену
Add-Computer -DomainName "company.local" -Credential (Get-Credential) -Restart

# С указанием конкретного контроллера домена
Add-Computer -DomainName "company.local" -Server "dc1.company.local" -Credential (Get-Credential) -Restart

# Без перезагрузки (для скриптов)
Add-Computer -DomainName "company.local" -Credential (Get-Credential) -Options SkipJoinOffline

# Проверка членства в домене
systeminfo | findstr /B /C:"Домен"
```

**Через командную строку:**
```cmd
# Присоединение к домену
netdom join %computername% /domain:company.local /userd:Administrator /passwordd:*

# Перезагрузка
shutdown /r /t 0

# Проверка
nltest /dsgetdc:company.local
```

### 3.4. Проверка членства в домене

**Через PowerShell:**
```powershell
# Проверка членства в домене
Test-ComputerSecureChannel -Verbose

# Получение информации о домене
Get-WmiObject -Class Win32_ComputerSystem | Select-Object Domain, PartOfDomain

# Проверка доверительных отношений
nltest /sc_query:company.local
```

**Через командную строку:**
```cmd

systeminfo | findstr /B /C:"Домен"

# Результат должен быть: company.local
```

## 4. Аутентификация доменных пользователей

### 4.1. Механизмы аутентификации

**Kerberos (основной протокол):**
- **Принцип работы:** Использует билеты (tickets) для аутентификации
- **Преимущества:** Безопасность, односеансовая аутентификация (SSO)
- **Компоненты:**
  - KDC (Key Distribution Center) — на контроллере домена
  - TGT (Ticket Granting Ticket) — начальный билет
  - Service Ticket — билет для доступа к сервису

**Процесс аутентификации Kerberos:**
1. Пользователь входит в систему
2. Клиент отправляет запрос на KDC
3. KDC проверяет учетные данные и выдает TGT
4. При доступе к ресурсу клиент запрашивает Service Ticket
5. Сервис проверяет Service Ticket и предоставляет доступ

**NTLM (устаревший, для совместимости):**
- **Принцип работы:** Challenge-response аутентификация
- **Недостатки:** Меньшая безопасность, нет SSO
- **Использование:** Для устаревших систем и приложений

### 4.2. Вход в систему под доменной учетной записью

**Для Linux-клиентов:**

**Вход через консоль:**
```bash
# Вход с полным именем пользователя
login Administrator@company.local

# Или с коротким именем (если настроено)
login company\\Administrator

# Проверка текущей сессии
whoami

# Пример вывода: company\\administrator
```

**Вход через SSH:**
```bash
# Подключение с доменной учетной записью
ssh Administrator@client.company.local

# С указанием домена
ssh company\\Administrator@client.company.local
```

**Проверка прав пользователя:**
```bash
# Проверка групп пользователя
id
groups

# Проверка контекста безопасности
id company\\Administrator
```

**Для Windows-клиентов:**

**Вход на рабочую станцию:**
1. На экране входа: `company\Administrator` или `Administrator@company.local`
2. Ввести пароль доменного пользователя
3. Система проверяет учетные данные через контроллер домена

**Вход через RDP:**
```powershell
# Подключение с доменной учетной записью
mstsc /v:client.company.local

# В диалоге входа указать: company\Administrator
```

### 4.3. Кэширование учетных данных

**Назначение кэширования:**
- Возможность входа при недоступности контроллера домена
- Ускорение процесса аутентификации
- Снижение нагрузки на контроллер домена

**Настройка в Linux (SSSD):**
```bash
# В файле /etc/sssd/sssd.conf добавить:
[sssd]
cache_credentials = True
cache_credentials_group_memberships_in_key = True

# Перезапуск SSSD
systemctl restart sssd

# Проверка кэша
ls -la /var/lib/sss/db/
```

**Настройка в Windows:**
```powershell
# Количество кэшированных входов (по умолчанию 10)
# Реестр: HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon
# Параметр: CachedLogonsCount

# Установка через PowerShell
Set-ItemProperty -Path "HKLM:\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon" -Name "CachedLogonsCount" -Value "10"
```

**Проверка кэшированных учетных данных:**
```bash
# Для Linux
lastlog

# Для Windows
klist sessions
```

### 4.4. Односеансовая аутентификация (SSO)

**Принцип работы SSO:**
- Пользователь входит в систему один раз
- Получает Kerberos тикет (TGT)
- При доступе к ресурсам используется существующий тикет
- Нет необходимости повторного ввода пароля

**Настройка SSO в Linux:**
```bash
# Проверка наличия тикета
klist

# Получение тикета вручную (если нужно)
kinit Administrator

# Настройка PAM для автоматического получения тикета
# В /etc/pam.d/system-auth добавить:
auth optional pam_krb5.so use_first_pass
```

**Настройка SSO в Windows:**
- Автоматически работает при входе в домен
- Настройки в групповых политиках:
  - Конфигурация компьютера → Политики → Настройки Windows → Параметры безопасности → Локальные политики → Параметры безопасности
  - "Клиент Kerberos: Поддержка утверждений, составных удостоверений и журнала аудита"

## 5. Управление доменными сессиями

### 5.1. Профили пользователей

**Типы профилей:**

**Локальный профиль:**
- Создается при первом входе локального пользователя
- Хранится в: `/home/username` (Linux) или `C:\Users\username` (Windows)
- Не перемещается между компьютерами

**Доменный профиль:**
- Создается при первом входе доменного пользователя
- Аналогичная структура, но с доменными настройками
- Может быть перемещаемым (Roaming Profile)

**Перемещаемый профиль (Roaming Profile):**
- Хранится на сетевом ресурсе
- Загружается при входе на любой компьютер домена
- Синхронизируется при выходе

**Обязательный профиль (Mandatory Profile):**
- Только для чтения
- Изменения не сохраняются
- Используется для киосков, общих рабочих мест

### 5.2. Домашние каталоги

**Настройка в Linux:**

**Автоматическое создание:**
```bash
# Настройка через oddjobd (уже настроено ранее)
systemctl status oddjobd

# Проверка создания домашнего каталога
ls -la /home/
ls -la /home/Administrator/
```

**Настройка через SSSD:**
```bash
# В /etc/sssd/sssd.conf:
[domain/company.local]
override_homedir = /home/%u
fallback_homedir = /home/%u@%d

# %u - имя пользователя
# %d - домен
```

**Сетевые домашние каталоги:**
```bash
# Монтирование сетевого ресурса как домашнего каталога
# В /etc/sssd/sssd.conf:
override_homedir = /mnt/home/%u

# Настройка автоматического монтирования
# В /etc/fstab:
//server.company.local/home /mnt/home cifs credentials=/etc/samba/credentials,uid=%U,gid=%G 0 0
```

**Настройка в Windows:**

**Через Active Directory:**
1. Открыть "Active Directory Users and Computers"
2. Свойства пользователя → Профиль
3. Путь к домашнему каталогу: `\\server\users\%username%`
4. При входе каталог автоматически монтируется

**Через групповые политики:**
- Конфигурация пользователя → Настройки → Настройка панели управления → Маппинг дисков
- Создать маппинг для домашней папки

### 5.3. Скрипты входа/выхода

**Назначение скриптов:**
- Автоматическая настройка окружения
- Маппинг сетевых дисков
- Установка принтеров
- Применение настроек безопасности

**Скрипт входа для Linux:**
```bash
# Создание скрипта /etc/profile.d/domain-login.sh
cat > /etc/profile.d/domain-login.sh << 'EOF'
#!/bin/bash

# Проверка доменного пользователя
if echo "$USER" | grep -q "@"; then
    echo "Добро пожаловать, доменный пользователь $USER"
    
    # Маппинг сетевого диска (пример)
    if [ -d "/mnt/home/$USER" ]; then
        ln -sf /mnt/home/$USER/Documents ~/Documents
    fi
fi
EOF

# Сделать исполняемым
chmod +x /etc/profile.d/domain-login.sh
```

**Скрипт выхода для Linux:**
```bash
# В /etc/pam.d/system-auth добавить:
session optional pam_exec.so /usr/local/bin/domain-logout.sh

# Скрипт /usr/local/bin/domain-logout.sh:
#!/bin/bash
# Очистка временных файлов
rm -rf /tmp/krb5cc_*
```

**Скрипты для Windows (PowerShell):**
```powershell
# Скрипт входа login.ps1
Write-Host "Добро пожаловать, $env:USERNAME"

# Маппинг сетевого диска
New-PSDrive -Name "H" -PSProvider FileSystem -Root "\\server\users\$env:USERNAME" -Persist

# Установка принтера
Add-Printer -ConnectionName "\\printserver\printer1"

# Применение через групповые политики:
# Конфигурация пользователя → Настройки → Сценарии → Вход в систему
```

### 5.4. Перемещаемые профили

**Настройка в Windows:**

**Через Active Directory:**
1. Свойства пользователя → Профиль
2. Путь: `\\server\profiles\%username%`
3. При входе профиль загружается с сервера
4. При выходе изменения сохраняются

**Через групповые политики:**
- Конфигурация компьютера → Политики → Административные шаблоны → Система → Профили пользователей
- "Назначить обязательный профиль для всех пользователей"

**Оптимизация перемещаемых профилей:**
- Исключение больших файлов (видео, музыка)
- Перенаправление папок (Документы, Рабочий стол)
- Использование Folder Redirection вместо Roaming Profiles

## 6. Диагностика и устранение неполадок

### 6.1. Типичные проблемы при присоединении

## 7. Безопасность при добавлении узлов

### 7.1. Безопасное присоединение к домену

**Использование защищенных соединений:**
```bash
# Принудительное использование LDAPS (LDAP over SSL)
# В /etc/sssd/sssd.conf:
ldap_id_use_start_tls = False
ldap_tls_reqcert = demand

# Использование сертификатов
# Импортировать сертификат CA
cp ca-cert.crt /etc/pki/tls/certs/
update-ca-trust extract
```

**Ограничение прав на присоединение:**
```bash
# Создание специальной учетной записи для присоединения
samba-tool user create joinaccount password123

# Добавление в группу "Account Operators"
samba-tool group addmembers "Account Operators" joinaccount

# Ограничение прав через делегирование
# В Active Directory Users and Computers:
# - Делегировать управление → Создать пользовательское задание
# - Разрешить присоединение компьютеров к домену
```

**Использование одноразовых паролей:**
```bash
# Создание компьютера с одноразовым паролем
samba-tool computer create PC01 --computer-password=TempPass123

# При присоединении использовать этот пароль
realm join company.local -U Administrator --computer-ou="OU=Computers,DC=company,DC=local"
```

### 7.2. Аудит событий присоединения

**Включение аудита в Samba AD:**
```bash
# В /etc/samba/smb.conf добавить:
[global]
   audit:enable = yes
   audit:file = /var/log/samba/audit.log
   audit:prefix = %u|%I|%m

# Перезапуск Samba
systemctl restart samba-ad-dc

# Проверка логов
tail -f /var/log/samba/audit.log
```

**Аудит в Windows:**
```powershell
# Включение аудита событий входа
auditpol /set /subcategory:"Logon" /success:enable /failure:enable

# Включение аудита управления учетными записями
auditpol /set /subcategory:"Computer Account Management" /success:enable /failure:enable

# Просмотр событий
Get-WinEvent -LogName Security | Where-Object {$_.Id -eq 4720 -or $_.Id -eq 4741} | Select-Object -First 10
```

**Мониторинг подозрительной активности:**
```bash
# Проверка неудачных попыток входа
grep "authentication failed" /var/log/secure

# Проверка множественных попыток
grep "krb5kdc: PREAUTHENTICATION FAILED" /var/log/krb5kdc.log

# Мониторинг присоединений компьютеров
grep "computer" /var/log/samba/audit.log | grep "create"
```

### 7.3. Безопасность доменных учетных записей

**Политики паролей:**
```bash
# Просмотр политик паролей
samba-tool domain passwordsettings show

# Настройка политик
samba-tool domain passwordsettings set --complexity=on
samba-tool domain passwordsettings set --min-pwd-length=12
samba-tool domain passwordsettings set --max-pwd-age=42
```

**Блокировка учетных записей:**
```bash
# Настройка блокировки
samba-tool domain passwordsettings set --account-lockout-threshold=5
samba-tool domain passwordsettings set --account-lockout-duration=30
```

**Мониторинг активности:**
```bash

who

grep "authentication failure" /var/log/secure | wc -l
```
