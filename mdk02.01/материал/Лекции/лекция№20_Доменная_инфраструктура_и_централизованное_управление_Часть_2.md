## 1. Создание домена Samba AD на ALT Linux

### 1.1. Подготовка системы

**Требования к системе:**
- ALT Linux Server (версия 10.x или новее)
- Минимум 2 ГБ ОЗУ (рекомендуется 4 ГБ)
- 20 ГБ свободного места на диске
- Статический IP-адрес
- Настроенный DNS (или будет настроен автоматически)

**Предварительная настройка:**
```bash
# Обновление системы
apt-get update
apt-get upgrade -y

# Настройка статического IP-адреса (пример для eth0)
nmcli connection modify eth0 ipv4.addresses 192.168.1.1/24
nmcli connection modify eth0 ipv4.gateway 192.168.1.254
nmcli connection modify eth0 ipv4.dns "192.168.1.1;8.8.8.8"
nmcli connection modify eth0 ipv4.method manual
nmcli connection up eth0

# Проверка сетевого подключения
ip addr show eth0
ping -c 4 8.8.8.8
```

### 1.2. Установка пакетов Samba AD

**Установка необходимых пакетов:**
```bash
# Установка пакетов Samba AD
apt-get install samba-dc samba-dc-mitrekrb5 winbind libnss-winbind libpam-winbind -y

# Проверка установленных пакетов
rpm -qa | grep samba
```

**Остановка и отключение стандартных служб Samba:**
```bash
# Остановка служб, которые могут конфликтовать
systemctl stop smbd nmbd winbind

# Отключение автозапуска
systemctl disable smbd nmbd winbind

# Маскировка служб (опционально)
systemctl mask smbd nmbd
```

### 1.3. Provision домена

**Интерактивная настройка домена:**
```bash
# Запуск интерактивного режима provision
samba-tool domain provision --use-rfc2307 --interactive

# Пример ввода параметров:
# Realm: COMPANY.LOCAL
# Domain: COMPANY
# Server Role: dc
# Backend: SAMBA_INTERNAL
# Host Name: DC1
# Administrator password: ComplexPassword123!
```

**Автоматическая настройка домена:**
```bash
# Автоматический provision с указанием всех параметров
samba-tool domain provision \
  --realm=COMPANY.LOCAL \
  --domain=COMPANY \
  --adminpass='ComplexPassword123!' \
  --server-role=dc \
  --dns-backend=SAMBA_INTERNAL \
  --use-rfc2307

# Проверка созданной конфигурации
cat /usr/local/samba/etc/smb.conf
```

### 1.4. Настройка DNS и запуск служб

**Настройка DNS:**
```bash
# Samba AD автоматически настраивает DNS сервер
# Проверка конфигурации DNS
cat /usr/local/samba/private/dns.keytab

# Настройка системы на использование локального DNS
echo "nameserver 127.0.0.1" > /etc/resolv.conf
echo "nameserver 8.8.8.8" >> /etc/resolv.conf

# Проверка разрешения имен
nslookup company.local
```

**Запуск службы Samba AD:**
```bash
# Запуск службы
systemctl start samba-ad-dc

# Включение автозапуска
systemctl enable samba-ad-dc

# Проверка статуса
systemctl status samba-ad-dc

# Проверка прослушиваемых портов
netstat -tuln | grep -E ':(53|88|135|139|389|445|464|636)'
```

### 1.5. Проверка работоспособности домена

**Базовые проверки:**
```bash
# Проверка информации о домене
samba-tool domain info

# Проверка DNS записей
samba-tool dns query localhost company.local @ A

# Проверка пользователей
samba-tool user list

# Проверка групп
samba-tool group list

# Проверка контроллеров домена
samba-tool fsmo show
```

**Тестирование аутентификации:**
```bash
# Получение Kerberos тикета
kinit Administrator

# Проверка тикета
klist

# Проверка доступа к LDAP
ldapsearch -H ldap://localhost -D "Administrator@COMPANY.LOCAL" -W -b "dc=company,dc=local" "(objectclass=*)"
```

## 2. Основные команды управления доменом

### 2.1. Управление пользователями

**Создание пользователей:**
```bash
# Создание нового пользователя
samba-tool user create username password123

# Создание пользователя с дополнительными параметрами
samba-tool user create \
  --username=john.doe \
  --password='SecurePass123!' \
  --userou="OU=Users,DC=company,DC=local" \
  --given-name="John" \
  --surname="Doe" \
  --email-address="john.doe@company.local"

# Проверка создания
samba-tool user show john.doe
```

**Изменение пользователей:**
```bash
# Сброс пароля
samba-tool user setpassword username --newpassword='NewPassword123!'

# Блокировка учетной записи
samba-tool user disable username

# Разблокировка учетной записи
samba-tool user enable username

# Удаление пользователя
samba-tool user delete username
```

**Просмотр информации о пользователях:**
```bash
# Список всех пользователей
samba-tool user list

# Поиск пользователя
samba-tool user show username

# Поиск пользователей по шаблону
samba-tool user list | grep -i "admin"
```

### 2.2. Управление группами

**Создание групп:**
```bash
# Создание новой группы
samba-tool group add "Group Name"

# Создание группы с указанием OU
samba-tool group add \
  --name="Sales Team" \
  --groupou="OU=Groups,DC=company,DC=local" \
  --description="Sales Department Group"

# Проверка создания
samba-tool group show "Sales Team"
```

**Управление членством in группах:**
```bash
# Добавление пользователя в группу
samba-tool group addmembers "Group Name" username

# Добавление нескольких пользователей
samba-tool group addmembers "Domain Users" user1,user2,user3

# Удаление пользователя из группы
samba-tool group removemembers "Group Name" username

# Просмотр членов группы
samba-tool group listmembers "Domain Admins"
```

**Просмотр групп:**
```bash
# Список всех групп
samba-tool group list

# Поиск группы
samba-tool group show "Group Name"

# Поиск групп пользователя
samba-tool user listgroups username
```

### 2.3. Управление компьютерами

**Добавление компьютеров в домен:**
```bash
# Создание учетной записи компьютера
samba-tool computer create PC01

# Создание с указанием OU
samba-tool computer create \
  --computername=WORKSTATION01 \
  --computerou="OU=Computers,DC=company,DC=local"

# Создание с одноразовым паролем
samba-tool computer create PC02 --computer-password='TempPass123'
```

**Управление компьютерами:**
```bash
# Просмотр компьютеров
samba-tool computer list

# Информация о конкретном компьютере
samba-tool computer show PC01

# Удаление компьютера
samba-tool computer delete PC01
```

### 2.4. Управление подразделениями (OU)

**Создание подразделений:**
```bash
# Создание OU через ldif
cat > create_ou.ldif << EOF
dn: OU=Users,DC=company,DC=local
objectClass: top
objectClass: organizationalUnit
ou: Users

dn: OU=Groups,DC=company,DC=local
objectClass: top
objectClass: organizationalUnit
ou: Groups

dn: OU=Computers,DC=company,DC=local
objectClass: top
objectClass: organizationalUnit
ou: Computers
EOF

# Импорт OU
ldbmodify -H /usr/local/samba/private/sam.ldb create_ou.ldif

# Проверка создания
samba-tool domain schema show | grep -i "ou"
```

### 2.5. Управление групповыми политиками (GPO)

**Создание и управление GPO:**
```bash
# Создание новой GPO
samba-tool gpo create "Test GPO"

# Просмотр всех GPO
samba-tool gpo listall

# Привязка GPO к подразделению
samba-tool gpo setlink "OU=Users,DC=company,DC=local" {GPO-ID}

# Удаление привязки GPO
samba-tool gpo delliink "OU=Users,DC=company,DC=local" {GPO-ID}

# Удаление GPO
samba-tool gpo del {GPO-ID}
```

### 2.6. Управление доменом

**Информация о домене:**
```bash
# Общая информация о домене
samba-tool domain info

# Информация о контроллере домена
samba-tool domain controller info

# Проверка FSMO ролей
samba-tool fsmo show

# Перемещение FSMO роли
samba-tool fsmo set --role=pdc --server=new_dc_name
```

**Настройка политик паролей:**
```bash
# Просмотр текущих настроек
samba-tool domain passwordsettings show

# Настройка минимальной длины пароля
samba-tool domain passwordsettings set --min-pwd-length=12

# Настройка сложности паролей
samba-tool domain passwordsettings set --complexity=on

# Настройка срока действия пароля
samba-tool domain passwordsettings set --max-pwd-age=42

# Настройка блокировки учетных записей
samba-tool domain passwordsettings set --account-lockout-threshold=5
```

## 3. Добавление Linux-клиента в домен

### 3.1. Подготовка Linux-клиента

**Установка необходимых пакетов:**
```bash
# Для ALT Linux
apt-get update
apt-get install realmd sssd sssd-tools adcli samba-common-tools krb5-user oddjob oddjob-mkhomedir -y

# Проверка установленных пакетов
rpm -qa | grep -E "(realmd|sssd|adcli|krb5)"
```

**Настройка сети и DNS:**
```bash
# Настройка DNS на контроллер домена
nmcli connection modify eth0 ipv4.dns "192.168.1.1"
nmcli connection modify eth0 ipv4.ignore-auto-dns yes
nmcli connection up eth0

# Проверка разрешения имен
nslookup company.local
ping dc1.company.local
```

**Синхронизация времени:**
```bash
# Установка и настройка NTP
apt-get install ntp -y
systemctl enable --now ntp

# Настройка на синхронизацию с контроллером домена
echo "server dc1.company.local iburst" >> /etc/ntp.conf
systemctl restart ntp

# Проверка синхронизации
ntpq -p
```

### 3.2. Присоединение к домену

**Проверка доступности домена:**
```bash
# Обнаружение домена
realm discover company.local

# Пример вывода:
# company.local
#   type: kerberos
#   realm-name: COMPANY.LOCAL
#   domain-name: company.local
#   configured: no
#   server-software: samba
#   client-software: sssd
```

**Присоединение к домену:**
```bash
# Присоединение с вводом пароля
realm join company.local -U Administrator

# Автоматическое присоединение (небезопасно)
echo "password" | realm join company.local -U Administrator --stdin

# Проверка членства в домене
realm list

# Пример вывода:
# company.local
#   type: kerberos
#   realm-name: COMPANY.LOCAL
#   domain-name: company.local
#   configured: kerberos-member
#   server-software: samba
#   client-software: sssd
```

### 3.3. Настройка SSSD

**Автоматическая настройка SSSD:**
```bash
# SSSD автоматически создает конфигурацию при присоединении
# Проверка созданного файла
cat /etc/sssd/sssd.conf

# Установка правильных прав
chmod 600 /etc/sssd/sssd.conf
chown root:root /etc/sssd/sssd.conf
```

**Ручная настройка SSSD (при необходимости):**
```bash
# Резервное копирование текущей конфигурации
cp /etc/sssd/sssd.conf /etc/sssd/sssd.conf.backup

# Редактирование конфигурации
cat > /etc/sssd/sssd.conf << EOF
[sssd]
domains = company.local
config_file_version = 2
services = nss, pam

[domain/company.local]
id_provider = ad
access_provider = ad
chpass_provider = ad
ldap_id_mapping = True
fallback_homedir = /home/%u@%d
override_homedir = /home/%u
ldap_schema = rfc2307
enumerate = False
cache_credentials = True
use_fully_qualified_names = False
ldap_tls_cacertdir = /etc/pki/tls/certs
ldap_tls_cacert = /etc/pki/tls/certs/ca-bundle.crt
krb5_realm = COMPANY.LOCAL
ldap_kerberos_realm = COMPANY.LOCAL
EOF

# Перезапуск SSSD
systemctl restart sssd
```

**Настройка автоматического создания домашних каталогов:**
```bash
# Включение службы oddjobd
systemctl enable --now oddjobd

# Настройка PAM
authconfig --enablemkhomedir --update

# Или вручную
cat >> /etc/pam.d/system-auth << EOF
session required pam_mkhomedir.so skel=/etc/skel/ umask=0022
EOF
```

## 4. Добавление Windows-клиента в домен

### 4.1. Настройка сети

**Настройка статического IP:**
```powershell
# Через PowerShell (от администратора)
New-NetIPAddress -InterfaceAlias "Ethernet" -IPAddress 192.168.1.100 -PrefixLength 24 -DefaultGateway 192.168.1.1

Set-DnsClientServerAddress -InterfaceIndex 1 -ServerAddresses 192.168.1.1
```

**Проверка подключения:**
```powershell
# Проверка связи с контроллером домена
ping dc1.company.local

# Проверка разрешения имен
nslookup company.local

# Проверка доступности служб
Test-NetConnection -ComputerName dc1.company.local -Port 389
Test-NetConnection -ComputerName dc1.company.local -Port 88
```

### 4.2. Присоединение к домену

**Через графический интерфейс:**
1. Система → Дополнительные параметры системы
2. Имя компьютера → Изменить
3. Является членом домена: company.local
4. Ввести учетные данные администратора домена
5. Перезагрузить компьютер

**Через PowerShell:**
```powershell
# Присоединение к домену
Add-Computer -DomainName "company.local" -Credential (Get-Credential) -Restart

# С указанием контроллера домена
Add-Computer -DomainName "company.local" -Server "dc1.company.local" -Credential (Get-Credential) -Restart

# Без перезагрузки
Add-Computer -DomainName "company.local" -Credential (Get-Credential) -Options SkipJoinOffline

# Проверка членства
systeminfo | findstr /B /C:"Домен"
```

**Через командную строку:**
```cmd
# Присоединение к домену
netdom join %computername% /domain:company.local /userd:Administrator /passwordd:*

# Перезагрузка
shutdown /r /t 0

# Проверка
nltest /dsgetdc:company.local
```

### 4.3. Проверка членства в домене

**Через PowerShell:**
```powershell

```

## 5. Аутентификация доменных пользователей

### 5.1. Механизмы аутентификации

**Kerberos (основной протокол):**
- **Принцип работы:** Использует билеты (tickets) для аутентификации
- **Преимущества:** Безопасность, односеансовая аутентификация (SSO)
- **Компоненты:**
  - KDC (Key Distribution Center) — на контроллере домена
  - TGT (Ticket Granting Ticket) — начальный билет
  - Service Ticket — билет для доступа к сервису

**Процесс аутентификации Kerberos:**
1. Пользователь входит в систему
2. Клиент отправляет запрос на KDC
3. KDC проверяет учетные данные и выдает TGT
4. При доступе к ресурсу клиент запрашивает Service Ticket
5. Сервис проверяет Service Ticket и предоставляет доступ

**NTLM (устаревший, для совместимости):**
- **Принцип работы:** Challenge-response аутентификация
- **Недостатки:** Меньшая безопасность, нет SSO
- **Использование:** Для устаревших систем и приложений

### 5.2. Вход в систему под доменной учетной записью

**Для Linux-клиентов:**

**Вход через консоль:**
```bash
# Вход с полным именем пользователя
login Administrator@company.local

# Или с коротким именем (если настроено)
login company\\Administrator

# Проверка текущей сессии
whoami

# Пример вывода: company\\administrator
```

**Вход через SSH:**
```bash
# Подключение с доменной учетной записью
ssh Administrator@client.company.local

# С указанием домена
ssh company\\Administrator@client.company.local
```

**Проверка прав пользователя:**
```bash
# Проверка групп пользователя
id
groups

# Проверка контекста безопасности
id company\\Administrator
```

**Для Windows-клиентов:**

**Вход на рабочую станцию:**
1. На экране входа: `company\Administrator` или `Administrator@company.local`
2. Ввести пароль доменного пользователя
3. Система проверяет учетные данные через контроллер домена

**Вход через RDP:**
```powershell
# Подключение с доменной учетной записью
mstsc /v:client.company.local

# В диалоге входа указать: company\Administrator
```

### 5.3. Кэширование учетных данных

**Назначение кэширования:**
- Возможность входа при недоступности контроллера домена
- Ускорение процесса аутентификации
- Снижение нагрузки на контроллер домена

**Настройка в Linux (SSSD):**
```bash
# В файле /etc/sssd/sssd.conf добавить:
[sssd]
cache_credentials = True
cache_credentials_group_memberships_in_key = True

# Перезапуск SSSD
systemctl restart sssd

# Проверка кэша
ls -la /var/lib/sss/db/
```

**Настройка в Windows:**
```powershell
# Количество кэшированных входов (по умолчанию 10)
# Реестр: HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon
# Параметр: CachedLogonsCount

# Установка через PowerShell
Set-ItemProperty -Path "HKLM:\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon" -Name "CachedLogonsCount" -Value "10"
```

### 5.4. Односеансовая аутентификация (SSO)

**Принцип работы SSO:**
- Пользователь входит в систему один раз
- Получает Kerberos тикет (TGT)
- При доступе к ресурсам используется существующий тикет
- Нет необходимости повторного ввода пароля

**Настройка SSO в Linux:**
```bash
# Проверка наличия тикета
klist

# Получение тикета вручную (если нужно)
kinit Administrator

# Настройка PAM для автоматического получения тикета
# В /etc/pam.d/system-auth добавить:
auth optional pam_krb5.so use_first_pass
```

**Настройка SSO в Windows:**
- Автоматически работает при входе в домен
- Настройки в групповых политиках:
  - Конфигурация компьютера → Политики → Настройки Windows → Параметры безопасности → Локальные политики → Параметры безопасности
  - "Клиент Kerberos: Поддержка утверждений, составных удостоверений и журнала аудита"

## 6. Настройка входа под доменными учетными записями

### 6.1. Профили пользователей

**Типы профилей:**

**Локальный профиль:**
- Создается при первом входе локального пользователя
- Хранится в: `/home/username` (Linux) или `C:\Users\username` (Windows)
- Не перемещается между компьютерами

**Доменный профиль:**
- Создается при первом входе доменного пользователя
- Аналогичная структура, но с доменными настройками
- Может быть перемещаемым (Roaming Profile)

**Перемещаемый профиль (Roaming Profile):**
- Хранится на сетевом ресурсе
- Загружается при входе на любой компьютер домена
- Синхронизируется при выходе

**Обязательный профиль (Mandatory Profile):**
- Только для чтения
- Изменения не сохраняются
- Используется для киосков, общих рабочих мест

### 6.2. Домашние каталоги

**Настройка в Linux:**

**Автоматическое создание:**
```bash
# Настройка через oddjobd (уже настроено ранее)
systemctl status oddjobd

# Проверка создания домашнего каталога
ls -la /home/
ls -la /home/Administrator/
```

**Настройка через SSSD:**
```bash
# В /etc/sssd/sssd.conf:
[domain/company.local]
override_homedir = /home/%u
fallback_homedir = /home/%u@%d

# %u - имя пользователя
# %d - домен
```

**Сетевые домашние каталоги:**
```bash
# Монтирование сетевого ресурса как домашнего каталога
# В /etc/sssd/sssd.conf:
override_homedir = /mnt/home/%u

# Настройка автоматического монтирования
# В /etc/fstab:
//server.company.local/home /mnt/home cifs credentials=/etc/samba/credentials,uid=%U,gid=%G 0 0
```

**Настройка в Windows:**

**Через Active Directory:**
1. Открыть "Active Directory Users and Computers"
2. Свойства пользователя → Профиль
3. Путь к домашнему каталогу: `\\server\users\%username%`
4. При входе каталог автоматически монтируется

**Через групповые политики:**
- Конфигурация пользователя → Настройки → Настройка панели управления → Маппинг дисков
- Создать маппинг для домашней папки

### 6.3. Скрипты входа/выхода

**Назначение скриптов:**
- Автоматическая настройка окружения
- Маппинг сетевых дисков
- Установка принтеров
- Применение настроек безопасности

**Скрипт входа для Linux:**
```bash
# Создание скрипта /etc/profile.d/domain-login.sh
cat > /etc/profile.d/domain-login.sh << 'EOF'
#!/bin/bash

# Проверка доменного пользователя
if echo "$USER" | grep -q "@"; then
    echo "Добро пожаловать, доменный пользователь $USER"
    
    # Маппинг сетевого диска (пример)
    if [ -d "/mnt/home/$USER" ]; then
        ln -sf /mnt/home/$USER/Documents ~/Documents
    fi
fi
EOF

# Сделать исполняемым
chmod +x /etc/profile.d/domain-login.sh
```

**Скрипты для Windows (PowerShell):**
```powershell
# Скрипт входа login.ps1
Write-Host "Добро пожаловать, $env:USERNAME"

# Маппинг сетевого диска
New-PSDrive -Name "H" -PSProvider FileSystem -Root "\\server\users\$env:USERNAME" -Persist

# Установка принтера
Add-Printer -ConnectionName "\\printserver\printer1"

# Применение через групповые политики:
# Конфигурация пользователя → Настройки → Сценарии → Вход в систему
```
