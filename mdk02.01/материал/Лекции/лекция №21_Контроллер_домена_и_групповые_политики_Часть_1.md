## 1. Введение в групповые политики

### 1.1. Понятие групповых политик

**Групповая политика (Group Policy)** — это инфраструктура управления конфигурациями, позволяющая централизованно настраивать и контролировать параметры операционных систем, приложений и безопасности для пользователей и компьютеров в доменной инфраструктуре.

**Ключевые возможности групповых политик:**
- **Централизованное управление** — администрирование множества компьютеров и пользователей из единой точки
- **Автоматизация настроек** — массовое применение конфигураций без ручного вмешательства
- **Контроль безопасности** — принудительное применение политик безопасности
- **Стандартизация** — обеспечение единообразия конфигураций в организации
- **Гибкость** — возможность применения различных настроек к разным группам объектов

### 1.2. Назначение и области применения

**Основные сценарии использования групповых политик:**

1. **Управление безопасностью:**
   - Политики паролей (длина, сложность, срок действия)
   - Политики блокировки учетных записей
   - Настройки брандмауэра
   - Политики аудита событий

2. **Настройка рабочей среды:**
   - Настройка рабочего стола и меню "Пуск"
   - Ограничение доступа к элементам управления
   - Настройка сетевых подключений
   - Управление принтерами

3. **Развертывание программного обеспечения:**
   - Автоматическая установка приложений
   - Обновление и удаление программ
   - Управление лицензиями

4. **Перенаправление ресурсов:**
   - Перенаправление папок (Документы, Рабочий стол)
   - Сетевые диски
   - Домашние каталоги

5. **Выполнение сценариев:**
   - Сценарии входа/выхода пользователей
   - Сценарии запуска/останова компьютеров

### 1.3. Эволюция групповых политик

**История развития:**

| Версия Windows | Год | Ключевые нововведения |
|----------------|-----|----------------------|
| Windows NT 4.0 | 1996 | Первые системные политики (System Policies) |
| Windows 2000 | 2000 | Group Policy в Active Directory |
| Windows XP/Server 2003 | 2001-2003 | Расширенные возможности управления |
| Windows Vista/Server 2008 | 2006-2008 | ADMX-шаблоны, улучшенная фильтрация |
| Windows 7/Server 2008 R2 | 2009 | Групповые предпочтения (Group Policy Preferences) |
| Windows 8/Server 2012 | 2012 | Центральное хранилище, улучшенное управление |
| Windows 10/Server 2016+ | 2015+ | Облачные политики, современные ADMX |

**Эволюция в Samba:**
- **Samba 4.0+** (2012) — базовая поддержка Active Directory
- **Samba 4.2+** (2015) — начальная поддержка групповых политик
- **Samba 4.4+** (2016) — улучшенная поддержка GPO через `samba-tool gpo`
- **Samba 4.11+** (2019) — стабильная реализация основных функций GPO

## 2. Архитектура групповых политик

### 2.1. Объект групповой политики (GPO)

**Объект групповой политики (Group Policy Object, GPO)** — это виртуальный контейнер, содержащий набор настроек групповой политики, которые применяются к пользователям и компьютерам.

**Структура GPO:**

```
GPO
├── Компьютерные настройки (Computer Configuration)
│   ├── Политики (Policies)
│   │   ├── Настройки Windows (Windows Settings)
│   │   │   ├── Параметры безопасности (Security Settings)
│   │   │   ├── Сценарии (Scripts)
│   │   │   └── Параметры IP (IP Settings)
│   │   └── Административные шаблоны (Administrative Templates)
│   └── Настройки (Preferences)
│       ├── Настройки панели управления (Control Panel Settings)
│       └── Настройки Windows (Windows Settings)
│
└── Пользовательские настройки (User Configuration)
    ├── Политики (Policies)
    │   ├── Настройки Windows (Windows Settings)
    │   │   ├── Параметры безопасности (Security Settings)
    │   │   ├── Сценарии (Scripts)
    │   │   └── Перенаправление папок (Folder Redirection)
    │   └── Административные шаблоны (Administrative Templates)
    └── Настройки (Preferences)
        ├── Настройки панели управления (Control Panel Settings)
        └── Настройки Windows (Windows Settings)
```

**Характеристики GPO:**
- **Уникальный идентификатор (GUID)** — каждый GPO имеет глобально уникальный идентификатор
- **Версия** — номер версии для отслеживания изменений
- **Статус** — включен/отключен для компьютеров и/или пользователей
- **Владелец** — создатель GPO, имеющий полные права
- **Комментарий** — описание назначения политики

### 2.2. Хранение GPO

**Двухкомпонентная структура хранения:**

1. **Group Policy Container (GPC)**
   - Хранится в Active Directory (LDAP)
   - Содержит метаданные GPO:
     - Имя, GUID, версия
     - Статус (включен/отключен)
     - Ссылки на GPO
     - Списки управления доступом (ACL)
   - Реплицируется между всеми контроллерами домена

2. **Group Policy Template (GPT)**
   - Хранится в файловой системе папки SYSVOL
   - Содержит фактические настройки политики:
     - ADMX/ADM-файлы (шаблоны)
     - Конфигурационные файлы
     - Сценарии
     - Файлы для развертывания ПО
   - Путь: `\\domain.com\SYSVOL\domain.com\Policies\{GPO-GUID}`
   - Реплицируется через FRS или DFS-R

**Структура каталога SYSVOL:**
```
SYSVOL\domain.com\Policies\
├── {GPO-GUID-1}\
│   ├── Machine\
│   │   ├── Microsoft\Windows NT\SecEdit\
│   │   │   └── GptTmpl.inf
│   │   ├── Scripts\
│   │   │   ├── Startup\
│   │   │   └── Shutdown\
│   │   └── Preferences\
│   └── User\
│       ├── Microsoft\Windows NT\SecEdit\
│       │   └── GptTmpl.inf
│       ├── Scripts\
│       │   ├── Logon\
│       │   └── Logoff\
│       └── Preferences\
│
└── {GPO-GUID-2}\
    └── ...
```

### 2.3. Версии политик и механизмы обновления

**Версионирование GPO:**
- Каждая GPO имеет номер версии (Version Number)
- Версия увеличивается при каждом изменении
- Клиенты сравнивают локальную версию с версией на контроллере домена
- При несовпадении версий происходит обновление политик

**Механизмы обновления:**

1. **Фоновое обновление (Background Refresh)**
   - Периодическая проверка обновлений
   - Для компьютеров: каждые 90 минут (±30 минут)
   - Для пользователей: каждые 90 минут (±30 минут)
   - Некоторые политики (безопасность, установка ПО) применяются только при перезагрузке/входе

2. **Обновление при загрузке/входе**
   - Компьютерные политики применяются при загрузке системы
   - Пользовательские политики применяются при входе в систему
   - Критические политики безопасности требуют перезагрузки/входа

3. **Принудительное обновление**
   ```bash
   # На клиенте Windows
   gpupdate /force
   
   # На клиенте Linux с Samba
   /usr/sbin/gpupdate --force
   ```

**Кэширование политик:**
- Клиенты кэшируют политики локально
- При недоступности контроллера домена используются кэшированные политики
- Кэш хранится в реестре и файловой системе

## 3. Наследование и применение групповых политик

### 3.1. Иерархия применения политик

Групповые политики применяются в строго определенном порядке, создавая иерархию наследования:

```
1. Локальные политики (Local GPO)
   ↓
2. Политики сайта (Site-linked GPO)
   ↓
3. Политики домена (Domain-linked GPO)
   ↓
4. Политики подразделений (OU-linked GPO)
   ├── Родительское OU
   ↓
   ├── Дочернее OU
   ↓
   └── Вложенное OU
```

**Правило LSDOU (Local, Site, Domain, OU):**
1. **Local** — локальные политики компьютера
2. **Site** — политики, связанные с сайтом Active Directory
3. **Domain** — политики, связанные с доменом
4. **OU** — политики, связанные с подразделениями (от родительских к дочерним)

**Пример иерархии:**
```
Домен: company.local
├── Локальные политики компьютера
├── Политики домена
│   ├── Default Domain Policy
│   └── Corporate Security Policy
│
└── Подразделения (OU)
    ├── Servers OU
    │   ├── Server Security Policy
    │   └── Backup Policy
    │
    ├── Workstations OU
    │   ├── Workstation Security Policy
    │   └── Software Deployment Policy
    │
    └── Users OU
        ├── User Security Policy
        └── Desktop Policy
```

### 3.2. Правила наследования и приоритеты

**Основные принципы:**
1. **Последний применяемый GPO имеет наивысший приоритет**
   - При конфликте настроек побеждает последняя примененная
   - Порядок: Local → Site → Domain → OU (от родительского к дочернему)

2. **Наследование по умолчанию**
   - Дочерние объекты наследуют политики родительских OU
   - Политики домена наследуются всеми объектами домена

3. **Накопительный эффект**
   - Все применимые политики накапливаются
   - Если настройки не конфликтуют, они все применяются

**Пример разрешения конфликтов:**
```
Политика 1 (Domain):    Запретить панель управления
Политика 2 (OU Users):  Разрешить панель управления

Результат: Разрешить (Политика 2 имеет приоритет как более поздняя)
```

### 3.3. Блокирование наследования (Block Inheritance)

**Назначение:**
- Предотвращает наследование политик от родительских контейнеров
- Применяется на уровне OU, домена или сайта

**Особенности:**
- Блокирует ВСЕ политики от вышестоящих уровней
- Не блокирует политики, помеченные как "Принудительные" (Enforced)
- Может привести к непредсказуемым последствиям

**Пример использования:**
```
OU: Development
├── Block Inheritance: Включено
├── Наследуемые политики: НЕ применяются
└── Локальные политики OU: Применяются
```

**Команда для блокировки наследования в Samba:**
```bash
# Блокировать наследование для OU
samba-tool gpupdate --block-inheritance --ou="OU=Development,DC=company,DC=local"
```

### 3.4. Принудительное применение (Enforced/No Override)

**Назначение:**
- Гарантирует применение политики независимо от блокировки наследования
- Помечает GPO как "непереопределяемый"
- Применяется на уровне ссылки GPO

**Особенности:**
- Принудительные политики имеют наивысший приоритет
- Не могут быть переопределены другими политиками
- Используются для критически важных настроек безопасности

**Пример:**
```
GPO: Corporate Security Policy
├── Enforced: Да
├── Настройки:
│   ├── Минимальная длина пароля: 12 символов
│   ├── Сложность пароля: Включена
│   └── Блокировка учетной записи: 5 неудачных попыток
│
Результат: Эти настройки НЕ могут быть изменены другими GPO
```

**Команда для принудительного применения:**
```bash
# Сделать GPO принудительным
samba-tool gpupdate --enforced --gpo="Corporate Security Policy"
```

### 3.5. Фильтрация WMI и безопасность

**Фильтрация на основе безопасности (Security Filtering):**
- Ограничивает применение GPO определенными группами безопасности
- По умолчанию: "Authenticated Users" (все аутентифицированные пользователи)
- Можно указать конкретные группы, пользователей или компьютеры

**Пример настройки безопасности:**
```
GPO: Software Deployment
├── Security Filtering:
│   ├── Domain Computers (чтение)
│   ├── IT Admins (полный доступ)
│   └── Sales Department (чтение и применение)
│
Результат: GPO применяется только к компьютерам отдела продаж
```

**WMI-фильтрация (Windows Management Instrumentation):**
- Динамическое определение применимости GPO на основе характеристик системы
- Примеры условий:
  - Версия операционной системы
  - Объем оперативной памяти
  - Наличие определенного ПО
  - Сетевые настройки

**Пример WMI-фильтра:**
```sql
-- Применять только к Windows 10
SELECT * FROM Win32_OperatingSystem 
WHERE Caption LIKE "%Windows 10%"

-- Применять только к компьютерам с 8+ GB RAM
SELECT * FROM Win32_ComputerSystem 
WHERE TotalPhysicalMemory > 8000000000
```

**Проверка WMI-фильтра:**
```bash
# Тестирование WMI-фильтра
wbemtest
# Подключиться к namespace: root\cimv2
# Выполнить запрос
```

## 4. Порядок обработки групповых политик

### 4.1. Последовательность применения (LSDOU)

**Детальный процесс применения политик:**

```
1. Инициализация
   ├── Загрузка списка GPO из Active Directory
   ├── Проверка версий GPO
   └── Определение применимых политик

2. Применение компьютерных политик (при загрузке)
   ├── Локальные политики (Local GPO)
   ├── Политики сайта (Site-linked GPO)
   ├── Политики домена (Domain-linked GPO)
   └── Политики OU (OU-linked GPO)
       ├── Родительские OU
       └── Дочерние OU

3. Применение пользовательских политик (при входе)
   ├── Локальные политики (Local GPO)
   ├── Политики сайта (Site-linked GPO)
   ├── Политики домена (Domain-linked GPO)
   └── Политики OU (OU-linked GPO)
       ├── Родительские OU
       └── Дочерние OU

4. Обработка конфликтов
   ├── Определение приоритетов
   ├── Разрешение конфликтов настроек
   └── Финальное применение политик
```

**Временная диаграмма:**
```
Загрузка компьютера:
[BIOS/UEFI] → [Загрузчик] → [Ядро ОС] → [Компьютерные GPO] → [Службы] → [Экран входа]

Вход пользователя:
[Экран входа] → [Аутентификация] → [Пользовательские GPO] → [Рабочий стол]
```

### 4.2. Синхронная и асинхронная обработка

**Асинхронная обработка (по умолчанию):**
- Политики применяются без ожидания завершения
- Система не ждет завершения обработки политик
- Более быстрая загрузка/вход
- Возможны проблемы с зависимыми службами

**Синхронная обработка:**
- Система ждет завершения обработки политик
- Гарантирует применение политик до продолжения загрузки
- Медленнее, но надежнее
- Требуется для некоторых политик безопасности

**Настройка режима обработки:**
```
GPO: Computer Configuration → Administrative Templates → System → Group Policy
├── Configure user Group Policy background processing:
│   ├── Allow asynchronous processing (по умолчанию)
│   └── Require synchronous processing
│
└── Specify startup policy processing wait time:
    └── Wait time: 0-65535 секунд
```

### 4.3. Фоновое обновление и перезагрузка/вход в систему

**Политики, применяемые при фоновом обновлении:**
- Административные шаблоны (большинство)
- Политики безопасности (некоторые)
- Настройки реестра
- Перенаправление папок (некоторые)

**Политики, требующие перезагрузки/входа:**
- Установка/удаление программного обеспечения
- Перенаправление папок (первоначальное)
- Сценарии входа/запуска
- Политики безопасности (некоторые)
- Настройки IP

**Принудительное применение при следующем входе:**
```
GPO: Computer Configuration → Administrative Templates → System → Group Policy
├── Configure user Group Policy background processing:
│   └── Process even if the Group Policy objects have not changed: Включено
│
└── Group Policy refresh interval for computers:
    └── Refresh interval: 0-44640 минут
```

### 4.4. Особенности обработки в Samba DC

**Отличия от Windows AD:**
1. **Ограниченная поддержка GPO**
   - Не все типы политик поддерживаются
   - В основном политики безопасности и базовые настройки

2. **Инструменты управления**
   ```bash
   # Просмотр GPO
   samba-tool gpo listall
   
   # Создание GPO
   samba-tool gpo create "New Policy"
   
   # Привязка GPO to OU
   samba-tool gpo setlink "OU=Users,DC=domain,DC=com" "New Policy"
   
   # Применение политик
   samba-tool gpupdate --all
   ```

3. **Клиенты Linux**
   - Используют `sssd` и `realmd` для интеграции
   - Поддерживают базовые GPO через `gpupdate`
   - Ограниченная поддержка Windows-специфичных политик

4. **Репликация SYSVOL**
   - Использует DFS-R или rsync
   - Может требовать ручной настройки
   - Важно для корректного применения политик

## 5. Типы настроек групповых политик

### 5.1. Политики безопасности (Security Policies)

**Назначают параметры безопасности системы:**

1. **Политики учетных записей (Account Policies)**
   - Применяются только на уровне домена
   - Влияют на все учетные записи домена

   **Политика паролей (Password Policy):**
   ```
   ├── Минимальная длина пароля: 8 символов
   ├── Максимальный срок действия пароля: 42 дня
   ├── Минимальный срок действия пароля: 1 день
   ├── Требовать соответствия истории: 5 паролей
   └── Пароль должен соответствовать требованиям сложности: Включено
   ```

   **Политика блокировки учетной записи (Account Lockout Policy):**
   ```
   ├── Порог блокировки: 5 неудачных попыток
   ├── Длительность блокировки: 30 минут
   └── Сброс счетчика блокировки: 30 минут
   ```

   **Политика Kerberos (Kerberos Policy):**
   ```
   ├── Максимальный срок жизни билета пользователя: 10 часов
   ├── Максимальный срок жизни билета службы: 600 минут
   ├── Максимальная разница во времени: 5 минут
   └── Принудительная повторная аутентификация: Включена
   ```

2. **Локальные политики (Local Policies)**
   - **Политики аудита (Audit Policies)**
     - Аудит входа в систему
     - Аудит доступа к объектам
     - Аудит использования привилегий
     - Аудит управления учетными записями

   - **Назначение прав пользователя (User Rights Assignment)**
     - Вход в систему локально
     - Доступ к компьютеру из сети
     - Завершение работы системы
     - Изменение системного времени

   - **Параметры безопасности (Security Options)**
     - Переименование учетной записи администратора
     - Отключение учетной записи гостя
     - Требование подписи SMB
     - Настройки UAC

### 5.2. Политики аудита (Audit Policies)

**Типы аудита:**

| Событие | Успех | Неудача | Пример использования |
|---------|-------|---------|---------------------|
| Вход в систему | ✓ | ✓ | Отслеживание входов пользователей |
| Доступ к объектам | ✓ | ✓ | Контроль доступа к файлам |
| Использование привилегий | ✓ | ✓ | Мониторинг действий администраторов |
| Управление учетными записями | ✓ | ✓ | Отслеживание создания/удаления пользователей |
| Изменение политик | ✓ | ✓ | Контроль изменений политик безопасности |
| Перезагрузка системы | ✓ | ✓ | Отслеживание перезагрузок |

**Пример настройки аудита:**
```

├── Audit account logon events: Success, Failure
├── Audit account management: Success, Failure
├── Audit directory service access: Success, Failure
├── Audit logon events: Success, Failure
├── Audit object access: Success, Failure
├── Audit policy change: Success, Failure
├── Audit privilege use: Success, Failure
├── Audit process tracking: Success, Failure
├── Audit system events: Success, Failure
└── Audit global object access SACL: Success, Failure
```

**Просмотр журнала аудита:**

```bash
# На Windows
eventvwr.msc → Windows Logs → Security

cat /var/log/samba/log.*

```

### 5.3. Политики программного обеспечения (Software Installation)

**Методы развертывания:**

1. **Назначение (Assign)**
   - Приложение устанавливается автоматически
   - Для компьютеров: при загрузке
   - Для пользователей: при первом запуске приложения

2. **Публикация (Publish)**
   - Приложение доступно для установки через "Установка программ"
   - Только для пользователей
   - Не устанавливается автоматически

**Пример развертывания:**
```
GPO: User Configuration → Software Settings → Software installation
├── New → Package
│   ├── Путь: \\server\share\app.msi
│   ├── Метод развертывания: Assigned
│   ├── Удалить при выходе из области действия: Да
│   └── Переустановить при повреждении: Да
│
└── Свойства пакета:
    ├── Идентификатор продукта: {GUID}
    ├── Версия: 1.0.0
    └── Язык: 1049 (Russian)
```

**Требования:**
- MSI-пакеты (Windows Installer)
- Сетевая папка с доступом на чтение
- Соответствие архитектуры (x86/x64)

### 5.4. Политики перенаправления папок (Folder Redirection)

**Назначение:**
- Перемещение папок профиля пользователя на сетевой ресурс
- Централизованное резервное копирование
- Роуминг пользовательских данных

**Перенаправляемые папки:**
- Документы (Documents)
- Изображения (Pictures)
- Музыка (Music)
- Видео (Videos)
- Рабочий стол (Desktop)
- Загрузки (Downloads)
- Контакты (Contacts)
- Избранное (Favorites)
- Ссылки (Links)
- Сохраненные игры (Saved Games)
- Поиск (Searches)

**Варианты настройки:**
```
GPO: User Configuration → Policies → Windows Settings → Folder Redirection
├── Documents → Properties
│   ├── Setting: Basic - Redirect everyone's folder to the same location
│   ├── Target folder location: Create a folder for each user under the root path
│   ├── Root path: \\server\users$\%USERNAME%\Documents
│   ├── Policy Removal:
│   │   ├── Leave the folder in its new location when policy is removed
│   │   └── Redirect the folder back to the userprofile location when policy is removed
│   └── Settings:
│       ├── Grant the user exclusive rights to the folder: Да
│       ├── Move the contents of Documents to the new location: Да
│       └── Also apply redirect policy to child folders: Нет
```

**Преимущества:**
- Централизованное хранение данных
- Автоматическое резервное копирование
- Снижение объема передаваемых data при роуминге профилей
- Повышение производительности входа

**Недостатки:**
- Зависимость от сетевого подключения
- Увеличение нагрузки на файловый сервер
- Возможные проблемы с доступом при недоступности сервера

### 5.5. Сценарии (Scripts)

**Типы сценариев:**

1. **Сценарии запуска/останова компьютера (Startup/Shutdown Scripts)**
   - Выполняются в контексте системы
   - Запуск: при загрузке компьютера
   - Остановка: при выключении компьютера

2. **Сценарии входа/выхода пользователя (Logon/Logoff Scripts)**
   - Выполняются в контексте пользователя
   - Вход: при аутентификации пользователя
   - Выход: при выходе пользователя из системы

**Пример сценария входа:**
```batch
@echo off
REM Сценарий входа пользователя
REM Маппинг сетевых дисков
net use H: \\server\home$\%USERNAME%
net use S: \\server\software
net use P: \\server\public

REM Установка переменных окружения
setx PRINTER "\\server\printer1"

REM Запуск приложений
start "" "C:\Program Files\Company\App\app.exe"
```

**Пример сценария запуска:**
```bash
#!/bin/bash
# Сценарий запуска компьютера (Linux)
# Настройка сетевых параметров
ip addr add 192.168.1.100/24 dev eth0

# Запуск служб
systemctl start company-service

# Логирование
echo "Computer started at $(date)" >> /var/log/startup.log
```

**Настройка в GPO:**
```
GPO: Computer Configuration → Policies → Windows Settings → Scripts (Startup/Shutdown)
├── Startup → Properties
│   ├── Script Name: \\server\sysvol\domain\scripts\startup.bat
│   ├── Script Parameters: -silent
│   └── Run startup scripts asynchronously: Нет
│
└── Shutdown → Properties
    ├── Script Name: \\server\sysvol\domain\scripts\shutdown.bat
    └── Script Parameters: /cleanup
```

## 6. Особенности реализации в Samba DC

### 6.1. Поддерживаемые типы GPO в Samba

**Полностью поддерживаемые:**
- Политики безопасности (Security Policies)
- Политики учетных записей (Account Policies)
- Политики аудита (Audit Policies)
- Настройки реестра (Registry Settings)
- Сценарии (Scripts)

**Частично поддерживаемые:**
- Административные шаблоны (Administrative Templates)
- Перенаправление папок (Folder Redirection)
- Групповые предпочтения (Group Policy Preferences)

**Не поддерживаемые:**
- Развертывание программного обеспечения (Software Installation)
- Некоторые сложные настройки безопасности
- Политики на основе WMI-фильтров (ограниченно)

### 6.2. Отличия от Windows Active Directory

| Характеристика | Windows AD | Samba DC |
|----------------|------------|----------|
| **Поддержка GPO** | Полная | Базовая/частичная |
| **Инструменты управления** | GPMC, PowerShell | samba-tool gpo |
| **Редактирование GPO** | Графический интерфейс | Командная строка |
| **Центральное хранилище** | Автоматическое | Ручная настройка |
| **Репликация SYSVOL** | DFS-R/FRS | rsync/DFS-R |
| **Поддержка клиентов** | Все Windows | Windows + Linux |
| **Документация** | Обширная | Ограниченная |

### 6.3. Инструменты управления: samba-tool gpo

**Основные команды:**

```bash
# Просмотр всех GPO
samba-tool gpo listall

# Просмотр GPO по имени или GUID
samba-tool gpo list "GPO_NAME"
samba-tool gpo list {GPO_GUID}

# Создание новой GPO
samba-tool gpo create "New Policy" --displayname="New Policy"

# Удаление GPO
samba-tool gpo delete {GPO_GUID}

# Привязка GPO к OU
samba-tool gpo setlink "OU=Users,DC=domain,DC=com" "Policy Name"

# Удаление привязки GPO
samba-tool gpo dellink "OU=Users,DC=domain,DC=com" 1

# Просмотр привязок GPO
samba-tool gpo listlinks "OU=Users,DC=domain,DC=com"

# Включение/отключение GPO
samba-tool gpo setlinkstate "OU=Users,DC=domain,DC=com" 1 --enabled=yes
samba-tool gpo setlinkstate "OU=Users,DC=domain,DC=com" 1 --enabled=no

# Настройка принудительного применения
samba-tool gpo setlinkstate "OU=Users,DC=domain,DC=com" 1 --enforced=yes
samba-tool gpo setlinkstate "OU=Users,DC=domain,DC=com" 1 --enforced=no

# Блокировка наследования
samba-tool gpo setlinkstate "OU=Users,DC=domain,DC=com" --block-inheritance=yes
samba-tool gpo setlinkstate "OU=Users,DC=domain,DC=com" --block-inheritance=no

# Обновление политик
samba-tool gpupdate --all
samba-tool gpupdate --target=user
samba-tool gpupdate --target=computer

# Экспорт/импорт GPO
samba-tool gpo export {GPO_GUID} > policy.xml
samba-tool gpo import policy.xml

# Изменение настроек GPO
samba-tool gpo setconfig "Policy Name" --config="config.ini"
```

**Пример создания комплексной политики:**
```bash
# 1. Создание GPO
samba-tool gpo create "Workstation Security Policy"

# 2. Привязка к OU
samba-tool gpo setlink "OU=Workstations,DC=company,DC=local" "Workstation Security Policy"

# 3. Настройка параметров безопасности
samba-tool gpo setconfig "Workstation Security Policy" --config="security.conf"

# 4. Включение принудительного применения
samba-tool gpo setlinkstate "OU=Workstations,DC=company,DC=local" 1 --enforced=yes

# 5. Обновление политик на всех клиентах
samba-tool gpupdate --all
```

### 6.4. Совместимость с Windows-клиентами

**Поддерживаемые клиенты:**
- Windows 7 и новее
- Windows Server 2008 R2 и новее
- Windows 10/11 (с ограничениями)
- Windows Server 2016/2019/2022

**Особенности совместимости:**
1. **Базовые политики безопасности** — работают корректно
2. **Административные шаблоны** — требуют ADMX-файлов
3. **Сценарии** — работают с ограничениями
4. **Перенаправление папок** — работает с ограничениями
5. **Развертывание ПО** — не поддерживается

**Настройка клиентов:**
```powershell
# На Windows клиенте
# 1. Ввод в домен
Add-Computer -DomainName "company.local" -Credential "Administrator"

# 2. Проверка связи с контроллером домена
Test-ComputerSecureChannel -Verbose

# 3. Обновление политик
gpupdate /force

# 4. Просмотр примененных политик
gpresult /r
gpresult /h report.html
```

### 6.5. Ограничения и рекомендации

**Известные ограничения:**
1. **Неполная поддержка ADMX**
   - Не все административные шаблоны работают
   - Требуется ручная настройка некоторых параметров

2. **Ограниченная поддержка групповых предпочтений**
   - Не все типы предпочтений реализованы
   - Могут быть проблемы с совместимостью

3. **Отсутствие графического редактора GPO**
   - Только командная строка
   - Требует глубоких знаний

4. **Проблемы с репликацией SYSVOL**
   - Требует тщательной настройки
   - Возможны задержки репликации

**Рекомендации по использованию:**
1. **Тестирование в лабораторной среде**
   - Перед внедрением в production
   - Проверка совместимости с клиентами

2. **Документирование политик**
   - Ведение журнала изменений
   - Описание назначения каждой политики

3. **Регулярное резервное копирование**
   - Экспорт GPO
   - Резервное копирование SYSVOL

4. **Мониторинг применения политик**
   - Проверка журналов событий
   - Анализ отчетов gpresult

5. **Использование проверенных конфигураций**
   - Начинать с базовых политик
   - Постепенное усложнение

1. **Групповая политика (Group Policy)** — инфраструктура централизованного управления конфигурациями
2. **Объект групповой политики (GPO)** — контейнер с настройками политик
3. **Наследование** — механизм применения политик от родительских контейнеров к дочерним
4. **Приоритет** — порядок применения политик при конфликтах
5. **Блокирование наследования** — предотвращение наследования от вышестоящих уровней
6. **Принудительное применение** — гарантия применения политики независимо от блокировки
7. **Фильтрация** — ограничение применения политик на основе безопасности или WMI
8. **LSDOU** — порядок применения: Local, Site, Domain, OU
9. **SYSVOL** — общая папка для хранения GPO
10. **Административные шаблоны** — настраиваемые параметры реестра
