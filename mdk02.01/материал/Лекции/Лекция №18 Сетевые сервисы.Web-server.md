# Введение

## Понятие веб-сервера и его роль в современной инфраструктуре

Веб-сервер — это программное обеспечение, которое обрабатывает HTTP-запросы от клиентов (обычно веб-браузеров) и возвращает им HTTP-ответы, содержащие веб-страницы, изображения, файлы и другие ресурсы. Веб-серверы играют ключевую роль в современной ИТ-инфраструктуре, обеспечивая доступ к веб-приложениям, корпоративным порталам, системам управления контентом и многим другим сервисам.

**Основные функции веб-сервера:**
- Обработка HTTP-запросов от клиентов
- Отдача статического контента (HTML, CSS, JavaScript, изображения)
- Генерация динамического контента с помощью серверных языков программирования
- Управление подключениями и сессиями
- Логирование и мониторинг доступа
- Обеспечение безопасности соединений (SSL/TLS)

**Роль веб-сервера в корпоративной среде:**
- Хостинг корпоративных веб-сайтов и порталов
- Предоставление доступа к внутренним бизнес-приложениям
- Интеграция с базами данных и другими сервисами
- Балансировка нагрузки и обеспечение отказоустойчивости

## История развития веб-серверов

**Предшественники современных веб-серверов:**
- **1989 год** — Тим Бернерс-Ли разработал концепцию World Wide Web
- **1990 год** — создан первый веб-сервер CERN httpd
- **1995 год** — появление Apache HTTP Server, который быстро стал самым популярным веб-сервером

**Эволюция LAMP-стека:**
- **1995 год** — создание PHP (Personal Home Page)
- **1995 год** — выпуск MySQL версии 1.0
- **Конец 1990-х** — формирование классического LAMP-стека (Linux + Apache + MySQL + PHP)
- **2000-е годы** — массовое распространение LAMP для создания динамических веб-сайтов
- **2010-е годы** — появление альтернатив (Nginx, PostgreSQL) и развитие LAMP

**Современное состояние:**
- Apache остается одним из самых популярных веб-серверов
- PHP 7.x и 8.x обеспечивают высокую производительность
- MariaDB (форк MySQL) стал стандартом для многих дистрибутивов Linux
- LAMP-стек широко используется для хостинга WordPress, Drupal, Joomla и других CMS

## Принципы работы веб-сервера

### Архитектура HTTP-протокола

**HTTP (HyperText Transfer Protocol)** — протокол прикладного уровня для передачи гипертекстовых документов.

**Методы HTTP:**

| Метод | Назначение | Идемпотентность |
|-------|------------|-----------------|
| GET | Получение ресурса | Да |
| POST | Отправка данных на сервер | Нет |
| PUT | Замена ресурса | Да |
| DELETE | Удаление ресурса | Да |
| HEAD | Получение заголовков | Да |
| OPTIONS | Получение поддерживаемых методов | Да |
| PATCH | Частичное изменение ресурса | Нет |

**Коды состояния HTTP:**

- **1xx (Informational)** — информационные коды
- **2xx (Success)** — успешное выполнение запроса
  - 200 OK — запрос успешно выполнен
  - 201 Created — ресурс создан
  - 204 No Content — запрос выполнен, но контента нет
- **3xx (Redirection)** — перенаправление
  - 301 Moved Permanently — постоянное перенаправление
  - 302 Found — временное перенаправление
  - 304 Not Modified — ресурс не изменился
- **4xx (Client Error)** — ошибка клиента
  - 400 Bad Request — неверный запрос
  - 401 Unauthorized — требуется аутентификация
  - 403 Forbidden — доступ запрещен
  - 404 Not Found — ресурс не найден
- **5xx (Server Error)** — ошибка сервера
  - 500 Internal Server Error — внутренняя ошибка сервера
  - 502 Bad Gateway — неверный ответ от вышестоящего сервера
  - 503 Service Unavailable — сервис недоступен

**Структура HTTP-запроса:**
```
GET /index.html HTTP/1.1
Host: www.example.com
User-Agent: Mozilla/5.0
Accept: text/html
```

**Структура HTTP-ответа:**
```
HTTP/1.1 200 OK
Date: Mon, 31 Mar 2026 10:00:00 GMT
Server: Apache/2.4.41
Content-Type: text/html; charset=UTF-8
Content-Length: 1234

<!DOCTYPE html>
<html>
...
</html>
```

### Модульная архитектура Apache

Apache HTTP Server использует модульную архитектуру, что позволяет подключать и отключать функциональность по мере необходимости.

**MPM (Multi-Processing Modules):**

1. **prefork** — процессная модель
   - Каждый запрос обрабатывается отдельным процессом
   - Высокая стабильность
   - Большое потребление памяти
   - Совместимость с не-потокобезопасными модулями (mod_php)

2. **worker** — гибридная модель
   - Множество процессов, каждый с несколькими потоками
   - Меньшее потребление памяти
   - Высокая производительность
   - Требует потокобезопасных модулей

3. **event** — событийная модель
   - Оптимизирована для обработки большого количества keep-alive соединений
   - Наименьшее потребление памяти
   - Наивысшая производительность
   - Рекомендуется для высоконагруженных серверов

**Основные модули Apache:**

| Модуль | Назначение |
|--------|------------|
| mod_ssl | Поддержка SSL/TLS |
| mod_rewrite | Переписывание URL |
| mod_proxy | Проксирование запросов |
| mod_auth_basic | Базовая аутентификация |
| mod_dir | Обработка директорий |
| mod_mime | Определение типов контента |
| mod_log_config | Логирование |
| mod_headers | Управление заголовками |
| mod_deflate | Сжатие контента |
| mod_expires | Управление кэшированием |

**Загрузка модулей:**
```bash
# Включение модуля
a2enmod ssl
a2enmod rewrite

# Отключение модуля
a2dismod status

# Перезагрузка Apache
systemctl restart apache2
```

### Обработка PHP-скриптов

**Способы интеграции PHP с Apache:**

1. **mod_php (Apache module)**
   - PHP встроен в Apache как модуль
   - Высокая производительность для небольших нагрузок
   - Простота настройки
   - Потребляет больше памяти

2. **PHP-FPM (FastCGI Process Manager)**
   - PHP работает как отдельный сервис
   - Лучшая масштабируемость
   - Меньшее потребление памяти
   - Рекомендуется для высоконагруженных систем

**Жизненный цикл PHP-запроса:**
```
1. Клиент отправляет HTTP-запрос
2. Apache получает запрос
3. Apache определяет, что запрос требует обработки PHP
4. Apache передает запрос PHP-интерпретатору
5. PHP выполняет скрипт
6. PHP возвращает результат Apache
7. Apache отправляет ответ клиенту
```

**Конфигурация PHP (php.ini):**
```ini
; Основные параметры
memory_limit = 256M
max_execution_time = 30
upload_max_filesize = 64M
post_max_size = 64M

; Безопасность
display_errors = Off
log_errors = On
error_log = /var/log/php/error.log

; Расширения
extension=mysqli
extension=gd
extension=mbstring
```

### Основы работы с базами данных

**Реляционные базы данных:**
- Данные организованы в таблицы
- Таблицы связаны между собой
- Использование SQL для работы с данными

**Основные операции SQL (CRUD):**
- **Create** — INSERT (создание записей)
- **Read** — SELECT (чтение записей)
- **Update** — UPDATE (обновление записей)
- **Delete** — DELETE (удаление записей)

**Пример SQL-запроса:**
```sql
-- Создание таблицы
CREATE TABLE users (
    id INT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50) NOT NULL,
    email VARCHAR(100) NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Вставка данных
INSERT INTO users (username, email) VALUES ('admin', 'admin@example.com');

-- Выборка данных
SELECT * FROM users WHERE id = 1;

-- Обновление данных
UPDATE users SET email = 'new@example.com' WHERE id = 1;

-- Удаление данных
DELETE FROM users WHERE id = 1;
```

**Подключение PHP к MySQL:**
```php
<?php
// Подключение через MySQLi
$mysqli = new mysqli("localhost", "user", "password", "database");

// Подключение через PDO
$pdo = new PDO("mysql:host=localhost;dbname=database", "user", "password");
?>
```

## Компоненты LAMP-стека

### Apache HTTP Server

**Структура конфигурационных файлов в ALT Linux:**
```
/etc/apache2/
├── apache2.conf          # Основной конфигурационный файл
├── ports.conf            # Настройка портов

├── mods-available/       # Доступные модули
├── mods-enabled/         # Включенные модули
├── sites-available/      # Доступные виртуальные хосты
└── sites-enabled/        # Включенные виртуальные хосты
```

**Основные директории:**
- `/etc/apache2/` — конфигурационные файлы
- `/var/www/html/` — корневая директория по умолчанию
- `/var/log/apache2/` — логи Apache
- `/usr/lib/apache2/modules/` — модули Apache

### PHP (Hypertext Preprocessor)

**Особенности языка PHP:**
- Встраивается в HTML-код
- Выполняется на стороне сервера
- Динамическая типизация
- Широкие возможности для работы с базами данных

**Базовый синтаксис:**
```php
<?php
// Переменные
$name = "World";

// Условные операторы
if ($name == "World") {
    echo "Hello, World!";
}

// Циклы
for ($i = 0; $i < 10; $i++) {
    echo $i;
}

// Функции
function greet($name) {
    return "Hello, $name!";
}

echo greet("User");
?>
```

**Встроенные функции для работы с БД:**
```php
<?php
// MySQLi
$mysqli = new mysqli("localhost", "user", "password", "database");
$result = $mysqli->query("SELECT * FROM users");
while ($row = $result->fetch_assoc()) {
    echo $row['username'];
}

// PDO
$pdo = new PDO("mysql:host=localhost;dbname=database", "user", "password");
$stmt = $pdo->query("SELECT * FROM users");
while ($row = $stmt->fetch()) {
    echo $row['username'];
}
?>
```

**Архитектура СУБД:**

- **Библиотеки** — libmysqlclient

**Типы данных MySQL:**

| Тип | Описание |

| VARCHAR(n) | Строка переменной длины |
| TEXT | Длинный текст |

**Пользователи и привилегии:**

## Конфигурирование Apache

### Структура конфигурационных файлов

**Основной конфигурационный файл /etc/apache2/apache2.conf:**
```apache
# Базовая конфигурация
ServerRoot "/etc/apache2"
Timeout 300
KeepAlive On
MaxKeepAliveRequests 100
KeepAliveTimeout 5

# Пользователь и группа
User www-data
Group www-data

# Логирование
ErrorLog ${APACHE_LOG_DIR}/error.log
LogLevel warn
CustomLog ${APACHE_LOG_DIR}/access.log combined

# Включение конфигурационных файлов
IncludeOptional mods-enabled/*.load
IncludeOptional mods-enabled/*.conf
Include ports.conf
IncludeOptional conf-enabled/*.conf
IncludeOptional sites-enabled/*.conf
```

### Основные директивы конфигурации

**Настройка портов (/etc/apache2/ports.conf):**
```apache
Listen 80

<IfModule ssl_module>
    Listen 443
</IfModule>

<IfModule mod_gnutls.c>
    Listen 443
</IfModule>
```

**Настройка виртуального хоста по умолчанию (/etc/apache2/sites-available/000-default.conf):**
```apache
<VirtualHost *:80>
    ServerAdmin webmaster@localhost
    DocumentRoot /var/www/html
    
    ErrorLog ${APACHE_LOG_DIR}/error.log
    CustomLog ${APACHE_LOG_DIR}/access.log combined
</VirtualHost>
```

### Настройка виртуальных хостов

**1. Создание нового виртуального хоста:**
```bash
# Создание директории для сайта
mkdir -p /var/www/example.com/public_html

# Создание тестовой страницы
echo "<h1>Welcome to example.com</h1>" > /var/www/example.com/public_html/index.html

# Назначение прав
chown -R www-data:www-data /var/www/example.com
chmod -R 755 /var/www/example.com
```

**2. Создание конфигурации виртуального хоста:**
```apache
# /etc/apache2/sites-available/example.com.conf
<VirtualHost *:80>
    ServerAdmin admin@example.com
    ServerName example.com
    ServerAlias www.example.com
    DocumentRoot /var/www/example.com/public_html
    
    <Directory /var/www/example.com/public_html>
        Options -Indexes +FollowSymLinks
        AllowOverride All
        Require all granted
    </Directory>
    
    ErrorLog ${APACHE_LOG_DIR}/example.com-error.log
    CustomLog ${APACHE_LOG_DIR}/example.com-access.log combined
</VirtualHost>
```

**3. Включение виртуального хоста:**
```bash
# Включение сайта
a2ensite example.com.conf

# Перезагрузка Apache
systemctl reload apache2

# Проверка конфигурации
apache2ctl configtest
```

**4. Настройка нескольких сайтов:**
```bash
# Создание второго сайта
mkdir -p /var/www/site2.com/public_html
echo "<h1>Welcome to site2.com</h1>" > /var/www/site2.com/public_html/index.html

# Создание конфигурации
# /etc/apache2/sites-available/site2.com.conf
<VirtualHost *:80>
    ServerName site2.com
    ServerAlias www.site2.com
    DocumentRoot /var/www/site2.com/public_html
    
    <Directory /var/www/site2.com/public_html>
        AllowOverride All
        Require all granted
    </Directory>
</VirtualHost>

# Включение сайта
a2ensite site2.com.conf
systemctl reload apache2
```

## Конфигурирование PHP

**Расположение конфигурационных файлов PHP в ALT Linux:**
- Для Apache: `/etc/php/*/apache2/php.ini`

```ini
date.timezone = Asia/Yekaterinburg

; Сессии
session.save_path = "/var/lib/php/sessions"
session.cookie_httponly = 1
session.use_strict_mode = 1

; Расширения
extension=mysqli
extension=pdo_mysql
extension=gd
extension=mbstring
extension=xml
extension=curl
```

# Безопасность веб-сервера

## Основные угрозы

### SQL-инъекции

**Описание:** Атака, при которой злоумышленник внедряет вредоносный SQL-код в запросы к базе данных.

**Пример уязвимого кода:**
```php
// ОПАСНО! Уязвимо к SQL-инъекциям
$username = $_GET['username'];
$query = "SELECT * FROM users WHERE username = '$username'";
$result = $mysqli->query($query);
```

**Защищенный код:**
```php
// Безопасно! Использование подготовленных выражений
$stmt = $pdo->prepare("SELECT * FROM users WHERE username = ?");
$stmt->execute([$_GET['username']]);
$user = $stmt->fetch();
```

### XSS-атаки (Cross-Site Scripting)

**Описание:** Атака, при которой злоумышленник внедряет вредоносный JavaScript-код на страницы сайта.

**Пример уязвимого кода:**
```php
// ОПАСНО! Уязвимо к XSS
echo "<h1>" . $_GET['title'] . "</h1>";
```

**Защищенный код:**
```php
// Безопасно! Экранирование вывода
echo "<h1>" . htmlspecialchars($_GET['title'], ENT_QUOTES, 'UTF-8') . "</h1>";
```

### DDoS-атаки

**Описание:** Атака, направленная на отказ в обслуживании путем перегрузки сервера запросами.

**Методы защиты:**
- Настройка rate limiting в Apache
- Использование mod_evasive
- Настройка fail2ban
- Использование CDN и сервисов защиты от DDoS

## Методы защиты

### Настройка .htaccess

**Запрет доступа к определенным файлам:**
```apache
# /var/www/html/.htaccess

# Запрет доступа к скрытым файлам
<FilesMatch "^\.">
    Require all denied
</FilesMatch>

# Запрет доступа к конфигурационным файлам
<FilesMatch "\.(ini|log|conf|inc|dist)$">
    Require all denied
</FilesMatch>

# Защита от кликов по фрейму
Header always append X-Frame-Options SAMEORIGIN

# Защита от MIME-сниффинга
Header set X-Content-Type-Options "nosniff"

# Включение XSS-защиты
Header set X-XSS-Protection "1; mode=block"
```

### Ограничение доступа к директориям

**Настройка прав доступа:**
```apache
# /etc/apache2/conf-available/directory-security.conf

# Отключение листинга директорий

    Options -Indexes +FollowSymLinks

</Directory>

# Ограничение доступа к админ-панели
```

# Глоссарий терминов

- **Apache HTTP Server** — самый популярный открытый веб-сервер
- **PHP (Hypertext Preprocessor)** — серверный язык программирования для создания динамических веб-страниц
- **MySQL/MariaDB** — реляционная система управления базами данных
- **LAMP** — аббревиатура стека технологий: Linux, Apache, MySQL, PHP
- **HTTP (HyperText Transfer Protocol)** — протокол передачи гипертекста
- **SSL/TLS** — протоколы криптографической защиты соединения
- **Виртуальный хост** — возможность размещения нескольких сайтов на одном сервере
- **DocumentRoot** — корневая директория веб-сайта
- **.htaccess** — файл конфигурации Apache для настройки директорий
- **SQL-инъекция** — атака, внедряющая вредоносный SQL-код
- **XSS (Cross-Site Scripting)** — атака, внедряющая вредоносный JavaScript
- **CORS (Cross-Origin Resource Sharing)** — механизм разрешения跨доменных запросов
- **MPM (Multi-Processing Module)** — модуль многопоточной обработки Apache
- **mod_php** — модуль Apache для обработки PHP-скриптов
- **PHP-FPM** — FastCGI Process Manager для PHP
- **PDO (PHP Data Objects)** — интерфейс доступа к базам данных в PHP
- **Подготовленные выражения** — безопасный способ выполнения SQL-запросов
- **HSTS (HTTP Strict Transport Security)** — заголовок для принудительного использования HTTPS
- **WAF (Web Application Firewall)** — межсетевой экран для веб-приложений
