## Политика лицензирования, свободные лицензии

Политика лицензирования программного обеспечения определяет правила использования, распространения и модификации ПО. В мире свободного ПО (open-source) ключевыми являются свободные лицензии, которые гарантируют пользователям определенные свободы.

Основные свободы свободного ПО по определению Free Software Foundation (FSF):
- Свобода 0: Запуск программы для любых целей.
- Свобода 1: Изучение и модификация исходного кода.
- Свобода 2: Распространение копий.
- Свобода 3: Улучшение программы и распространение улучшенной версии.

Популярные свободные лицензии:
- **GPL (GNU General Public License)**: Копилефт-лицензия, требующая, чтобы производные работы также были свободными. Версии: GPL v2, v3.
- **MIT License**: Простая лицензия, позволяющая использовать, модифицировать и распространять код с минимальными ограничениями.
- **BSD License**: Аналогично MIT, с акцентом на академическое использование.
- **Apache License 2.0**: Включает патентную защиту и дополнительные условия.

Свободные лицензии способствуют сотрудничеству, инновациям и снижению зависимости от проприетарного ПО. В Linux-сообществе они являются основой экосистемы.

## История развития Unix/Linux

История Unix/Linux начинается в 1960-х годах. В 1969 году в Bell Labs (AT&T) Кен Томпсон и Деннис Ритчи разработали Unix как многозадачную ОС для компьютеров. Unix стал основой для многих систем, включая BSD (Berkeley Software Distribution).

В 1970-х годах Билл Гейтс основал Microsoft и разработал MS-DOS/Windows, которые стали доминирующими проприетарными ОС для персональных компьютеров, конкурируя с Unix.

В 1980-х годах Ричард Столман, недовольный закрытостью ПО, инициировал проект GNU (GNU's Not Unix) в 1983 году и основал Free Software Foundation (FSF). Цель — создать полностью свободную ОС, аналогичную Unix.

В 1991 году Линус Торвальдс, студент из Финляндии, анонсировал ядро Linux как свободный клон Unix. Linux быстро развивался благодаря сообществу, объединившись с инструментами GNU, что привело к полноценной ОС GNU/Linux.

Сегодня Linux — основа для серверов, мобильных устройств (Android) и суперкомпьютеров, конкурируя с Windows и macOS.

## Проект GNU

Проект GNU, запущенный Ричардом Столманом в 1983 году, направлен на создание полностью свободной операционной системы, совместимой с Unix. "GNU" — рекурсивный акроним "GNU's Not Unix".

Ключевые компоненты GNU:
- **GCC (GNU Compiler Collection)**: Компиляторы для языков C, C++, Fortran и др.
- **Bash**: Командная оболочка.
- **Emacs**: Текстовый редактор.
- **glibc**: Стандартная библиотека C.
- **GNU Coreutils**: Утилиты для работы с файлами, процессами и т.д.

GNU под лицензией GPL обеспечивает, что ПО остается свободным. Без GNU проект Linux остался бы лишь ядром; GNU предоставил экосистему инструментов.

Проект GNU повлиял на движение свободного ПО, вдохновив лицензии и сообщества.

## Основные ветви дистрибутивов Linux

Дистрибутивы Linux — готовые к установке сборки ОС на базе ядра Linux и инструментов GNU. Они классифицируются по происхождению и целям.

Международные ветви:
- **Debian**: Стабильный, ориентирован на свободное ПО.
- **Red Hat/Fedora**: Корпоративные решения (RHEL, CentOS).
- **SUSE**: Предпринимательские дистрибутивы (openSUSE).

Приоритетно российские дистрибутивы:
- **Альт Linux**: Семейство ОС от компании "Базальт СПО", ориентировано на российский рынок, с поддержкой отечественного ПО.
- **ROSA Linux**: Разработка компании ROSA Labs, совместим с RPM, популярна в России.
- **Astra Linux**: Специализирован для государственных структур, сертифицирован по требованиям ФСТЭК.
- **Calculate Linux**: Основан на Gentoo, фокус на десктоп и серверы.
- **RED OS**: Разработка "Ред Софт", ориентирован на госсектор.

Российские дистрибутивы адаптированы к локальным требованиям, включая поддержку кириллицы, интеграцию с отечественным ПО и соответствие стандартам безопасности.

## Краткая история семейства ОС «Альт»

Семейство ОС «Альт» развивается с 2001 года. Первоначально основано на Mandrake Linux, позже перешло на собственную базу. В 2008 году создана компания "Базальт СПО" для коммерческой поддержки.

Ключевые этапы:
- 2001: Запуск ALT Linux.
- 2004: Переход на пакетный менеджер APT-RPM.
- 2010-е: Разработка специализированных дистрибутивов для образования, бизнеса и госсектора.
- 2020-е: Интеграция с отечественными технологиями, фокус на безопасность и импортозамещение.

«Альт» позиционируется как альтернатива зарубежным ОС в России.

## Компания "Базальт СПО"

"Базальт СПО" — российская компания, основанная в 2008 году для разработки и поддержки семейства ОС «Альт». Специализируется на open-source решениях для бизнеса, образования и госсектора.

Деятельность:
- Разработка дистрибутивов Альт.
- Техническая поддержка и консалтинг.
- Участие в проекте Sisyphus.
- Интеграция с российским ПО.

Компания способствует развитию отечественной ИТ-инфраструктуры.

## Основные характеристики ОС «Альт»

ОС «Альт» — дистрибутивы на базе Linux с уникальными характеристиками:
- **Свободное ПО**: Полностью открытый исходный код.
- **Многоязычность**: Поддержка русского и других языков.
- **Пакетный менеджер APT-RPM**: Удобное управление ПО.
- **Безопасность**: Интеграция SELinux, поддержка шифрования.
- **Адаптация**: Для российских пользователей, включая отечественное ПО.
- **Стабильность**: Регулярные обновления и долгосрочная поддержка.

«Альт» подходит для серверов, рабочих станций и образования.

## Дистрибутивы семейства ОС «Альт»

Семейство включает несколько дистрибутивов:
- **ALT Linux Workstation**: Для рабочих станций, с графическим интерфейсом.
- **ALT Linux Server**: Серверная версия для предприятий.
- **ALT Linux Education**: Для образовательных учреждений, с предустановленным ПО.
- **ALT Linux Compact**: Легковесная версия для старого оборудования.
- **ALT Linux Rescue**: Система восстановления.

Каждый адаптирован под специфические задачи.

## Пакеты, репозитории, пакетный менеджер

В Linux ПО распространяется в виде пакетов — архивов с файлами и метаданными.

**Пакеты**: Содержат бинарные файлы, скрипты установки, зависимости.

**Репозитории**: Хранилища пакетов, локальные или удаленные (например, Sisyphus).

**Пакетный менеджер**: Инструмент для установки/обновления. В «Альт» — APT-RPM (apt-get, aptitude), основан на Debian APT, но работает с RPM-пакетами.

Команды: `apt-get install`, `apt-get update`.

Это обеспечивает удобное управление зависимостями и обновлениями.

## Проект Sisyphus

Sisyphus — основной репозиторий Альт Linux, постоянно обновляемый. Назван в честь мифа о Сизифе, символизируя непрерывную работу.

Особенности:
- Тысячи пакетов с исходниками.
- Ежедневные обновления.
- Для разработчиков и тестирования.
- Основа для стабильных дистрибутивов.

Участие в Sisyphus позволяет вносить вклад в развитие.

## Обзор окружений рабочего стола

Окружения рабочего стола (DE) — графические интерфейсы для взаимодействия с ОС.

Популярные:
- **GNOME**: Современный, минималистичный, стандарт в многих дистрибутивах.
- **KDE Plasma**: Функциональный, настраиваемый, с Plasma Desktop.
- **XFCE**: Легковесный, для слабых ПК.
- **LXDE/LXQt**: Минималистичные, быстрые.
- **MATE/Cinnamon**: Классические, производные от GNOME 2.

В «Альт» используются XFCE, KDE, GNOME в зависимости от дистрибутива. Выбор зависит от предпочтений пользователя.
