## Краткие теоретические сведения

### DNS (Domain Name System)
DNS — система доменных имен, предназначенная для преобразования доменных имен в IP-адреса и наоборот. DNS является распределенной базой данных, обеспечивающей удобное именование сетевых ресурсов.

### bind9 в ALT Linux
В ALT Linux 2.4 Master DNS-сервер bind9 работает в специальном chroot-окружении для повышения безопасности. Особенности:
- Named запускается в каталоге `/var/lib/bind` как корневом каталоге
- Главный файл настроек `/etc/named.conf` является символьной ссылкой
- Конфигурационные файлы находятся в `/var/lib/bind/etc/`
- Зоны хранятся в `/var/lib/bind/zone/`

### Типы DNS-записей
- **A** — IPv4 адрес
- **AAAA** — IPv6 адрес
- **CNAME** — псевдоним
- **MX** — почтовый сервер
- **NS** — DNS-сервер зоны
- **SOA** — начало полномочий зоны
- **PTR** — обратное разрешение

### 3. Настройка основных параметров DNS-сервера

#### 3.1. Редактирование options.conf
Откройте файл настроек:

```bash
nano /var/lib/bind/etc/options.conf
```

#### 3.2. Настройка параметров
Добавьте или измените следующие параметры:

```conf
# /var/lib/bind/etc/options.conf
options {
    directory "/var/lib/bind/zone";
    
    # Настройка вышестоящих DNS-серверов
    forwarders {
        8.8.8.8;      # Google DNS
    };
    
    # Ограничение доступа к серверу (только для подсети стенда)
    allow-query { 192.168.1.0/24; };
    allow-recursion { 192.168.1.0/24; };
    
    # Другие настройки
    listen-on { 192.168.1.3; };
    listen-on-v6 { any; };
    
};
```

#### 3.3. Проверка настроек
```bash
# Проверка настроек allow-query и allow-recursion
grep -A 5 "allow-" /var/lib/bind/etc/options.conf
```

#### 3.4. Применение изменений
```bash
# Обновление chroot-окружения
update_chrooted conf

# Проверка применения изменений
service bind status
```

### 4. Создание DNS-зон

#### 4.1. Создание прямой зоны sa.test

**4.1.1. Копирование пустого шаблона**
```bash
# Копирование файла empty для создания прямой зоны
cp /var/lib/bind/zone/empty /var/lib/bind/zone/sa.test
```

**4.1.2. Редактирование прямой зоны**
```bash
nano /var/lib/bind/zone/sa.test
```

**4.1.3. Заполнение прямой зоны**
```conf
# /var/lib/bind/zone/sa.test
$TTL    1D
@       IN      SOA     sa.test. root.sa.test. (
                                2023120101      ; serial
                                12H             ; refresh
                                1H              ; retry
                                1W              ; expire
                                1H              ; ncache
                        )
        IN      NS      srv.sa.test.
        IN      A       192.168.1.3

# DNS-сервер
srv     IN      A       192.168.1.3

# Другие серверы
isp     IN      A       192.168.1.1
cli     IN      A       192.168.1.200

# Псевдонимы
www     IN      CNAME   srv.sa.test.
ftp     IN      CNAME   srv.sa.test.
```

#### 4.2. Создание обратной зоны

**4.2.1. Копирование пустого шаблона**
```bash
# Копирование файла empty для создания обратной зоны
cp /var/lib/bind/zone/empty /var/lib/bind/zone/1.168.192.in-addr.arpa
```

**4.2.2. Редактирование обратной зоны**
```bash
nano /var/lib/bind/zone/1.168.192.in-addr.arpa
```

**4.2.3. Заполнение обратной зоны**
```conf
# /var/lib/bind/zone/1.168.192.in-addr.arpa
$TTL    1D
@       IN      SOA     sa.test. root.sa.test. (
                                2023120101      ; serial
                                12H             ; refresh
                                1H              ; retry
                                1W              ; expire
                                1H              ; ncache
                        )
        IN      NS      srv.sa.test.

# Обратные записи
1       IN      PTR     isp.sa.test.
3       IN      PTR     srv.sa.test.
200     IN      PTR     cli.sa.test.
```

#### 4.3. Добавление зон в конфигурацию

**4.3.1. Редактирование local.conf**
```bash
nano /var/lib/bind/etc/local.conf
```

**4.3.2. Добавление описания зон**
```conf
# /var/lib/bind/etc/local.conf
zone "sa.test" {
    type master;
    file "sa.test";
};

zone "1.168.192.in-addr.arpa" {
    type master;
    file "1.168.192.in-addr.arpa";
};
```

### 5. Проверка и активация DNS-сервера

#### 5.1. Проверка конфигурации

### 6. Настройка клиентов

#### 6.1. Настройка DNS-сервера на клиентах
На машине ISP настройте DNS-сервер:

```bash
# Настройка локального DNS-сервера
cat > /etc/net/ifaces/ens19/resolv.conf << 'EOF'
domain sa.test
nameserver 192.168.1.3
EOF
```
На машине srv настройте DNS-сервер:

```bash
# Настройка локального DNS-сервера
cat > /etc/net/ifaces/ens18/resolv.conf << 'EOF'
domain sa.test
nameserver 192.168.1.3
EOF
```

Для настройки DNS на машине cli нужно внести/изменить(если существуют) опции в настройки DHCP-сервера на машине **isp**:

```bash
option domain-name-servers 192.168.1.3;
option domain-name "sa.test";
```

На машине cli обновить настройки сети:
```bash
systemctl restart NetworkManager
```

#### 6.2. Проверка настроек
```bash
# Проверка настроек DNS
cat /etc/resolv.conf
```

### 9. Интеграция с DHCP (ограниченно)

**Важно:** В этой лабораторной работе не делается привязка DHCP и DNS через rndc-key, так как CLI получает IP-адрес по DHCP, но для DNS используется фиксированный адрес 192.168.1.200.

#### 9.1. Подготовка к будущей интеграции
Для будущей интеграции DHCP и DNS понадобится:
- Генерация rndc-key
- Настройка динамических обновлений зон
- Интеграция с DHCP-сервером

#### 9.2. Временное решение
Пока используется фиксированный IP-адрес для CLI:
```conf
# В прямой зоне
cli     IN      A       192.168.1.200

# В обратной зоне
200     IN      PTR     cli.sa.test.
```
