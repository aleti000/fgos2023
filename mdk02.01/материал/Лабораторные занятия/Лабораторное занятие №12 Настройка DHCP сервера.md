## Методические сведения

### DHCP (Dynamic Host Configuration Protocol)

DHCP — это сетевой протокол, предназначенный для автоматической настройки параметров сетевого подключения клиентских устройств в IP-сетях. DHCP позволяет централизованно управлять IP-адресным пространством и автоматически настраивать сетевые параметры клиентов.

**Преимущества DHCP:**
- Автоматическая настройка сетевых параметров без вмешательства пользователя
- Централизованное управление IP-адресным пространством
- Исключение конфликтов IP-адресов
- Эффективное использование IP-адресного пространства
- Упрощение администрирования сети

### Процесс работы DHCP (DORA)

**1. Discover (Поиск)**
Клиент рассылает широковещательный запрос DHCP Discover для поиска доступных DHCP-серверов в сети.

**2. Offer (Предложение)**
DHCP-сервер отвечает на запрос, предлагая IP-адрес и сетевые параметры.

**3. Request (Запрос)**
Клиент выбирает предложение и отправляет запрос на получение указанного IP-адреса.

**4. Acknowledge (Подтверждение)**
DHCP-сервер подтверждает выделение IP-адреса и передает все необходимые сетевые параметры.

### Архитектура лабораторной сети

На основе Лабораторного занятия №6, в данной работе используется следующая сетевая топология:

- **isp** (ALT Server) — шлюз с NAT, на котором будет установлен DHCP-сервер
- **server** — сервер, которому необходимо назначить постоянный IP-адрес
- **cli** — клиентская машина, получающая IP-адрес из пула

**Текущая конфигурация сети:**
- Подсеть: 192.168.1.0/24
- Шлюз по умолчанию: 192.168.1.1 (на isp)
- DNS-сервер: 8.8.8.8

### Резервирование IP-адресов

Резервирование позволяет закрепить конкретный IP-адрес за устройством с определенным MAC-адресом. Это необходимо для серверов и сетевого оборудования, которым требуется постоянный IP-адрес.

### Пул адресов

Пул адресов — это диапазон IP-адресов, которые DHCP-сервер может выдавать клиентам автоматически. В данной работе будет использоваться пул 192.168.1.150-192.168.1.250.

### Часть 3: Настройка DHCP-сервера

**3.1. Определение интерфейса для DHCP**

Определите, какой интерфейс на isp подключен к внутренней сети:

```bash
# Просмотр всех интерфейсов
ip addr show

# Определите интерфейс, имеющий IP-адрес 192.168.1.1
# Обычно это eth1 или enp0s8
```

**3.2. Настройка интерфейса в конфигурации DHCP**

Отредактируйте файл `/etc/sysconfig/dhcpd`:

```bash
# Откройте файл для редактирования
nano /etc/sysconfig/dhcpd

# Найдите строку INTERFACES="" и укажите интерфейс, подключенный к внутренней сети
# Например, если интерфейс называется eth1:
DHCPARGS="ens19"

# Сохраните файл (Ctrl+O, Enter) и закройте редактор (Ctrl+X)
```

**3.3. Резервное копирование конфигурационного файла**

```bash
# Скопируйте шаблон конфигурации 
cp /etc/dhcp/dhcpd.conf.sample /etc/dhcp/dhcpd.conf
```

**3.4. Создание новой конфигурации DHCP**

Создайте новый конфигурационный файл:

```bash
# Отредактируйте конфигурационный файл:
> /etc/dhcp/dhcpd.conf


# Глобальные параметры
option domain-name "lab.local";
option domain-name-servers 8.8.8.8;
default-lease-time 86400;
max-lease-time 172800;

# Авторитетный сервер
authoritative;

# Настройка подсети 192.168.1.0/24
subnet 192.168.1.0 netmask 255.255.255.0 {
    # Пул адресов для клиентов
    range 192.168.1.150 192.168.1.250;
    
    # Шлюз по умолчанию
    option routers 192.168.1.1;
    
    # Маска подсети
    option subnet-mask 255.255.255.0;
    
    # DNS-сервер
    option domain-name-servers 8.8.8.8;
    
    # Адрес широковещательной рассылки
    option broadcast-address 192.168.1.255;
}

# Резервирование IP-адреса для server
# Замените 00:11:22:33:44:55 на реальный MAC-адрес server
host server {
    hardware ethernet 00:11:22:33:44:55;
    fixed-address 192.168.1.10;
    option host-name "server";
}
```

## Возможные проблемы и их решение

**Проблема 1: DHCP-сервер не запускается**
- Проверьте синтаксис конфигурационного файла: `dhcpd -t -cf /etc/dhcp/dhcpd.conf`
- Проверьте права доступа к файлам
- Проверьте, что указанный интерфейс существует

**Проблема 2: Клиенты не получают IP-адреса**
- Проверьте, что DHCP-сервер слушает правильный интерфейс
- Проверьте, что клиенты настроены на получение IP по DHCP
- Проверьте сетевую связь между сервером и клиентами

**Проблема 3: Server не получает зарезервированный IP-адрес**
- Проверьте MAC-адрес в конфигурации
- Проверьте, что server запрашивает IP по DHCP
- Проверьте файл аренд на наличие ошибок

**Проблема 4: Нет доступа в интернет**
- Проверьте NAT на isp
- Проверьте маршрутизацию
- Проверьте DNS-серверы
