## Методические указания

### Теоретическая часть

#### Что такое файловый сервер?
**Файловый сервер** - это компьютер или устройство, предоставляющее файлы и ресурсы для хранения данных другим компьютерам в сети.

#### Сетевые протоколы файлового доступа

| Протокол | Описание | Преимущества | Недостатки | Типичное использование |
|----------|----------|--------------|------------|----------------------|
| **SMB/CIFS** | Server Message Block/Common Internet File System | Отличная интеграция с Windows, поддержка прав доступа | Сложнее в настройке, более высокие накладные расходы | Корпоративные сети, Windows-ориентированные среды |
| **NFS** | Network File System | Простота настройки, высокая производительность | Ограниченная поддержка прав доступа, слабая безопасность | Unix/Linux среды, высокопроизводительные вычисления |

#### Архитектура файлового сервера на базе LVM

```
Физические диски (/dev/vdb, /dev/vdc, /dev/vdd)
    ↓
Физические тома (PV) - созданные в Лаб №10
    ↓
Группа томов (VG) - vg_data
    ↓
Логические тома (LV):
├── lv_home (400M) - /mnt/lvm_home
├── lv_var (300M) - /mnt/lvm_var  
└── lv_tmp (500M) - /mnt/lvm_tmp
    ↓
Файловые системы (ext4)
    ↓
Сетевые протоколы (SMB/CIFS, NFS)
```

#### Преимущества использования LVM для файлового сервера
- **Гибкость** - возможность изменения размеров томов "на лету"
- **Надежность** - интеграция с RAID массивами
- **Производительность** - оптимизация под различные типы нагрузки
- **Управляемость** - централизованное управление дисковым пространством

#### Типы доступа к сетевым ресурсам
- **Публичный доступ** - без аутентификации (только для NFS)
- **Приватный доступ** - с аутентификацией пользователей
- **Гостевой доступ** - ограниченный доступ для гостей (SMB)

### Практическая часть

#### Шаг 1: Подготовка системы
Данная работа выполняется на ВМ server.

**Проверка существующей LVM инфраструктуры:**
```bash
# Проверка LVM томов (результаты Лаб №10)
pvdisplay
vgdisplay vg_data
lvdisplay

# Проверка смонтированных томов
df -h | grep lvm
mount | grep lvm

# Проверка существующих файловых систем
lsblk -f
```

**Ожидаемый результат:**
```
Физические тома: /dev/vdb, /dev/vdc, /dev/vdd
Группа томов: vg_data
Логические тома:
├── /dev/vg_data/lv_home (400M) → /mnt/lvm_home
├── /dev/vg_data/lv_var (300M) → /mnt/lvm_var
└── /dev/vg_data/lv_tmp (500M) → /mnt/lvm_tmp
```

#### Шаг 2: Установка и настройка Samba

**Установка Samba:**
```bash
# Обновление системы
apt-get update

# Установка Samba
apt-get install samba samba-common

# Проверка установки
systemctl status smbd
systemctl status nmbd
```

**Резервное копирование конфигурации:**
```bash
# Резервное копирование оригинального файла
cp /etc/samba/smb.conf /etc/samba/smb.conf.backup
```

#### Шаг 3: Настройка Samba для LVM томов

**Создание пользователей Samba:**
```bash
# Создание группы для пользователей
groupadd sambashare

# Создание пользователей системы
useradd -M -G sambashare user1
useradd -M -G sambashare user2

# Установка паролей
passwd user1
passwd user2

# Добавление пользователей в Samba
smbpasswd -a user1
smbpasswd -a user2

# Включение пользователей
smbpasswd -e user1
smbpasswd -e user2
```

**Настройка конфигурации Samba:**
```bash
# Редактирование конфигурационного файла
nano /etc/samba/smb.conf
```

**Пример конфигурации для LVM томов:**
```ini
[global]
   workgroup = WORKGROUP
   server string = Samba Server %v
   netbios name = ubuntu
   security = user
   map to guest = bad user
   dns proxy = no

# Доступ к LVM томам через SMB
[lvm_home]
   path = /mnt/lvm_home
   browsable = yes
   writable = yes
   guest ok = no
   read only = no
   valid users = @sambashare
   create mask = 0644
   directory mask = 0755

[lvm_var]
   path = /mnt/lvm_var
   browsable = yes
   writable = yes
   guest ok = yes
   read only = no
   create mask = 0644
   directory mask = 0755

```

#### Шаг 4: Установка и настройка NFS

**Установка NFS сервера:**
```bash
# Установка NFS сервера
apt-get install nfs-kernel-server nfs-common

# Проверка установки
systemctl status nfs-server
```

**Настройка экспорта LVM томов:**
```bash
# Редактирование файла экспорта NFS
nano /etc/exports
```

**Пример конфигурации экспорта:**
```
# Экспорт LVM томов через NFS
/mnt/lvm_tmp     192.168.1.0/24(rw,sync,no_root_squash,no_subtree_check)

# Для публичного доступа (только для lvm_tmp)
/mnt/lvm_tmp     *(ro,sync,no_root_squash,no_subtree_check)
```

**Применение конфигурации NFS:**
```bash
# Применение изменений
exportfs -ra

# Перезапуск NFS сервера
systemctl restart nfs-server
```

#### Шаг 5: Настройка прав доступа

**Настройка прав на LVM томах:**
```bash
# Установка прав для SMB доступа
setfacl -m u:user1:rwx /mnt/lvm_home
setfacl -m u:user2:rwx /mnt/lvm_home
setfacl -m u:user1:rwx /mnt/lvm_var
setfacl -m u:user2:rwx /mnt/lvm_var

# Проверка прав
getfacl /mnt/lvm_home
getfacl /mnt/lvm_var
```

**Настройка владельца для NFS:**
```bash
# Установка владельца для NFS томов
chown nobody:nogroup /mnt/lvm_tmp
chmod 755 /mnt/lvm_tmp
```

#### Шаг 6: Запуск и тестирование серверов

**Запуск служб:**
```bash
# Запуск Samba
systemctl restart smbd
systemctl restart nmbd
systemctl enable smbd
systemctl enable nmbd

# Запуск NFS
systemctl restart nfs-server
systemctl enable nfs-server
```

**Проверка конфигурации:**
```bash
# Проверка синтаксиса Samba
testparm

# Проверка экспорта NFS
exportfs -v
```

#### Шаг 7: Тестирование SMB доступа

**Тестирование из Linux:**
```bash
# Установка клиента (если не установлен)
apt-get install smbclient

# Проверка доступных ресурсов
smbclient -L localhost -U user1

# Подключение к ресурсу
smbclient //localhost/lvm_home -U user1

# Проверка доступа к гостевому ресурсу
smbclient //localhost/lvm_tmp -U guest
```


#### Шаг 8: Настройка клиента (ВМ cli)

**Подготовка клиентской машины:**
```bash
# Установка необходимых пакетов
apt-get update
apt-get install samba-client nfs-common cifs-utils

# Создание точек монтирования
mkdir -p /mnt/server_smb_home
mkdir -p /mnt/server_smb_tmp
mkdir -p /mnt/server_nfs_var
```

**Монтирование SMB ресурсов:**
```bash
# Монтирование SMB ресурсов
mount -t cifs //server_ip/lvm_home /mnt/server_smb_home -o username=user1,password=password1
mount -t cifs //server_ip/lvm_tmp /mnt/server_smb_tmp -o guest

# Проверка монтирования
df -h
mount | grep server
```

**Монтирование NFS ресурса:**
```bash
# Монтирование NFS ресурса
mount -t nfs server_ip:/mnt/lvm_var /mnt/server_nfs_var

# Проверка монтирования
df -h
mount | grep nfs
```

**Постоянное монтирование через /etc/fstab:**
```bash
# Редактирование fstab
nano /etc/fstab
```

**Пример записей в fstab:**
```
# SMB ресурсы
//server_ip/lvm_home /mnt/server_smb_home cifs username=user1,password=password1,uid=1000,gid=1000 0 0
//server_ip/lvm_tmp /mnt/server_smb_tmp cifs guest,uid=1000,gid=1000 0 0

# NFS ресурс
server_ip:/mnt/lvm_var /mnt/server_nfs_var nfs defaults 0 0
```

## Типичные проблемы и их решения

### Проблема 1: "Connection refused" при подключении к SMB
**Причина:** Служба Samba не запущена или брандмауэр блокирует порты
**Решение:**
```bash
# Проверить статус служб
systemctl status smbd
systemctl status nmbd

# Проверить брандмауэр
ufw status
```

### Проблема 2: "Permission denied" при доступе к NFS
**Причина:** Неправильные права на каталог или проблемы с экспортом
**Решение:**
```bash
# Проверить экспорт
exportfs -v

# Проверить права
ls -la /mnt/lvm_home

# Перезапустить NFS
systemctl restart nfs-server
```

### Проблема 3: "NT_STATUS_LOGON_FAILURE" в SMB
**Причина:** Неправильный пароль или пользователь не существует
**Решение:**
```bash
# Проверить пользователей
pdbedit -L

# Пересоздать пользователя
smbpasswd -a username
```

### Проблема 4: NFS ресурс не виден на клиенте
**Причина:** Неправильная конфигурация экспорта или сети
**Решение:**
```bash
# Проверить экспорт на сервере
exportfs -v

# Проверить сеть
ping client_ip
```

### Проблема 5: Ошибки при копировании файлов
**Причина:** Недостаточно прав или проблемы с файловой системой
**Решение:**
```bash
# Проверить права на запись
touch /mnt/lvm_home/test

# Проверить файловую систему
df -h /mnt/lvm_home
```

## Рекомендации по безопасности

1. **Аутентификация**
   - Используйте сильные пароли для Samba пользователей
   - Регулярно меняйте пароли пользователей
   - Ограничьте количество попыток входа

2. **Права доступа**
   - Принцип минимальных привилегий
   - Регулярно проверяйте права доступа
   - Используйте ACL для гибкого управления

3. **Брандмауэр**
   - Ограничьте доступ только необходимым IP-адресам
   - Регулярно проверяйте правила брандмауэра
   - Используйте VPN для удаленного доступа

4. **Логирование**
   - Включите детальное логирование
   - Регулярно просматривайте логи
   - Настройте оповещения о подозрительной активности

5. **Обновления**
   - Регулярно обновляйте Samba и NFS
   - Следите за security advisories
   - Тестируйте обновления перед установкой
