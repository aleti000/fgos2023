## Теоретическая часть

### 1. Система идентификации пользователей и групп

> 🔑 **Ключевой принцип:** В Linux идентификация происходит по числовым идентификаторам, а не по именам. Имена — лишь удобное представление для человека.

| Идентификатор | Назначение | Типичные диапазоны в ALT Linux |
|---------------|------------|-------------------------------|
| **UID** (User ID) | Уникальный идентификатор пользователя | Системные: 0–999<br>Обычные: 1000+ |
| **GID** (Group ID) | Уникальный идентификатор группы | Системные: 0–999<br>Обычные: 1000+ |
| **EUID** (Effective UID) | Идентификатор для проверки прав доступа | Может отличаться от реального UID при SUID |

**Системные файлы учетных записей:**

| Файл | Назначение | Доступ | Формат записи |
|------|------------|--------|---------------|
| `/etc/passwd` | Основная информация о пользователях | Чтение всем | `имя:x:UID:GID:описание:домашний_каталог:оболочка` |
| `/etc/shadow` | Хэшированные пароли и политики | Только root | `имя:хэш:дата_изменения:...` |
| `/etc/group` | Информация о группах | Чтение всем | `имя:x:GID:пользователи` |
| `/etc/gshadow` | Пароли групп и администраторы | Только root | `имя:пароль:админы:пользователи` |

**Примеры записей:**
```bash
# /etc/passwd
user1:x:1001:1001:Иван Иванов:/home/user1:/bin/bash

# /etc/shadow (пароль заменён на *)
user1:*:19705:0:99999:7:::

# /etc/group
developers:x:1001:user1,user2
```

> 💡 **Особенность ALT Linux:** В ALT Linux для управления пользователями также доступны графические утилиты `drakuser` и `drakconf`, но в данной работе фокус на консольных командах.

```
Схема аутентификации пользователя:
┌─────────────┐
│  Пользователь │  (вводит имя/пароль)
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ /etc/passwd │  ← Проверка существования пользователя
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ /etc/shadow │  ← Сравнение хэша введённого пароля
└──────┬──────┘
       │  (успешно/неуспешно)
       ▼
┌─────────────┐
│  Сессия     │  ← Назначение прав на основе UID/GID
└─────────────┘
```

### 2. Права доступа в Linux

#### Базовые права

| Право | Символ | Восьмеричное значение | Для файлов | Для каталогов |
|-------|--------|----------------------|------------|---------------|
| Чтение | `r` | 4 | Чтение содержимого | Просмотр списка файлов (`ls`) |
| Запись | `w` | 2 | Изменение содержимого | Создание/удаление файлов |
| Выполнение | `x` | 1 | Запуск как программы | Вход в каталог (`cd`) |

#### Категории пользователей

```
-rwxr-xr-- 1 user1 developers 1024 Jan 1 12:00 file.txt
 │ └─┬─┘└─┬─┘
 │   │   └─── other (остальные пользователи): r--
 │   └─────── group (группа developers): r-x
 └─────────── owner (владелец user1): rwx
```

#### Восьмеричное представление прав

> 🔑 **Важно:** Все права в Linux записываются в **восьмеричной системе счисления**. Ведущий ноль (`0`) указывает на восьмеричный формат.

| Символы | Восьмеричное число | Расшифровка |
|---------|-------------------|-------------|
| `rwx` | `07` | 4+2+1 — полный доступ |
| `rw-` | `06` | 4+2+0 — чтение и запись |
| `r-x` | `05` | 4+0+1 — чтение и выполнение |
| `r--` | `04` | 4+0+0 — только чтение |
| `---` | `00` | 0+0+0 — нет доступа |

**Полные права для объекта (владелец+группа+остальные):**
- `0777` — полный доступ для всех (`rwxrwxrwx`)
- `0755` — владелец: полный, группа/остальные: чтение+выполнение (`rwxr-xr-x`)
- `0644` — владелец: чтение+запись, группа/остальные: только чтение (`rw-r--r--`)

> 💡 **Примечание:** При использовании команды `chmod` ведущий ноль часто опускается для удобства (`chmod 755 file`), но строго технически корректная запись — `0755`.

#### Специальные права (для ознакомления)

| Право | Символ | Восьмеричное значение | Назначение | Пример использования |
|-------|--------|----------------------|------------|----------------------|
| **SUID** | `s` в позиции владельца | `4000` | Запуск с правами владельца | `passwd` — изменение `/etc/shadow` |
| **SGID** | `s` в позиции группы | `2000` | Запуск с правами группы / наследование группы в каталоге | Общие рабочие каталоги |
| **Sticky bit** | `t` в позиции других | `1000` | Только владелец может удалить свои файлы | `/tmp` — общий каталог |

**Комбинированные права:**
```bash
chmod 4755 file  # SUID + rwxr-xr-x → -rwsr-xr-x
chmod 2775 dir   # SGID + rwxrwxr-x → drwxrwsr-x
chmod 1777 tmp   # Sticky bit + rwxrwxrwx → drwxrwxrwt
```

### 3. Механизм маски создания прав (umask)

> 🔑 **Ключевой принцип:** `umask` — это **инверсная маска в восьмеричной системе**, которая определяет, какие права **запрещены** при создании новых файлов и каталогов.

#### Как работает umask

При создании нового объекта система применяет следующую формулу (все операции — в восьмеричной системе):

```
Фактические права = Базовые права - umask
```

Где:
- **Базовые права для файлов:** `0666` (`rw-rw-rw-`) — файлы по умолчанию не создаются с правом выполнения
- **Базовые права для каталогов:** `0777` (`rwxrwxrwx`)

#### Примеры расчёта (в восьмеричной системе)

| umask | Расчёт для файла (`0666` - `umask`) | Итоговые права файла | Расчёт для каталога (`0777` - `umask`) | Итоговые права каталога |
|-------|-----------------------------------|----------------------|--------------------------------------|--------------------------|
| `0022` | `0666` - `0022` = `0644` | `rw-r--r--` | `0777` - `0022` = `0755` | `rwxr-xr-x` |
| `0002` | `0666` - `0002` = `0664` | `rw-rw-r--` | `0777` - `0002` = `0775` | `rwxrwxr-x` |
| `0077` | `0666` - `0077` = `0600` | `rw-------` | `0777` - `0077` = `0700` | `rwx------` |
| `0007` | `0666` - `0007` = `0660` | `rw-rw----` | `0777` - `0007` = `0770` | `rwxrwx---` |

> 💡 **Важно:** Вычитание происходит строго в восьмеричной системе. Например:
> - `0666` - `0022` = `0644` (а не 644 в десятичной)
> - `0777` - `0077` = `0700` (а не 700 в десятичной)

#### Типичные значения umask

| Значение | Описание | Сценарий использования |
|----------|----------|------------------------|
| `0022` | Стандартное значение | Обычные пользователи — группа и остальные имеют только чтение |
| `0002` | Расширенный доступ для группы | Совместная работа в группе — группа имеет запись |
| `0077` | Максимальная изоляция | Конфиденциальные данные — только владелец имеет доступ |
| `0007` | Совместная работа без доступа «остальных» | Командная разработка — группа имеет полный доступ |

#### Где настраивается umask

| Уровень | Файл конфигурации | Применяется к |
|---------|-------------------|---------------|
| Системный | `/etc/login.defs` | Все новые пользователи |
| Для пользователя | `~/.bashrc`, `~/.profile` | Конкретный пользователь при входе |
| Текущая сессия | `umask значение` | Только текущая оболочка |

**Пример настройки в `/etc/login.defs`:**
```bash
# Стандартная маска для обычных пользователей
UMASK           022

# Маска для системных пользователей
USERGROUPS_ENAB yes
```

**Пример настройки в `~/.bashrc`:**
```bash
# Для совместной работы в группе
umask 002
```

#### Особенности вычислений в восьмеричной системе

При ручном расчёте помните:
- Цифры могут быть только от 0 до 7
- При вычитании возможен «заем» из старшего разряда:
```
  0666
- 0077
------
  0600  (из второго разряда «занимаем» 1 → 6 становится 16₈, 16₈ - 7 = 7₈, но с учётом заема получаем 0)
  ```

> 💡 **Практический совет:** Для проверки расчётов используйте команду:
> ```bash
> printf "%04o\n" $((8#666 - 8#$(umask)))
> ```
> Где `8#` указывает на восьмеричную систему счисления.

### 4. Основные команды управления учетными записями

| Команда | Назначение | Пример использования |
|---------|------------|----------------------|
| `useradd` | Создание пользователя | `useradd -m -g developers -s /bin/bash user1` |
| `usermod` | Изменение параметров | `usermod -aG testers user1` (добавить в доп. группу) |
| `userdel` | Удаление пользователя | `userdel -r user1` (с домашним каталогом) |
| `passwd` | Установка пароля | `passwd user1` |
| `groupadd` | Создание группы | `groupadd -g 2001 developers` |
| `groupmod` | Изменение группы | `groupmod -n newname oldname` |
| `groupdel` | Удаление группы | `groupdel developers` |
| `id` | Просмотр UID/GID | `id user1` |
| `groups` | Просмотр групп пользователя | `groups user1` |
| `chmod` | Изменение прав | `chmod 0755 file` или `chmod u+x file` |
| `chown` | Изменение владельца | `chown user1:developers file` |
| `chgrp` | Изменение группы | `chgrp developers file` |
| `umask` | Просмотр/установка маски | `umask` (просмотр), `umask 002` (установка) |
| `getfacl` | Просмотр расширенных прав (ACL) | `getfacl file` |

> 💡 **Шпаргалка для студентов:**  
> `↑↓` — история команд | `Tab` — автодополнение | `Ctrl+C` — прервать выполнение | `man команда` — справка  
> **Важно:** При записи прав всегда помните — это восьмеричные числа (`0755`, а не десятичное 755).

### Задание 2: Создание групп и пользователей ⭐ Базовое

1. Создайте группу `developers` с GID 2001:
   ```bash
   sudo groupadd -g 2001 developers
   ```

2. Создайте группу `testers` с GID 2002:
   ```bash
   sudo groupadd -g 2002 testers
   ```

3. Создайте пользователя `user1`:
   ```bash
   sudo useradd -m -u 3001 -g developers -s /bin/bash user1
   ```
   Где:
   - `-m` — создать домашний каталог
   - `-u 3001` — задать UID
   - `-g developers` — основная группа
   - `-s /bin/bash` — оболочка

4. Создайте пользователя `user2`:
   ```bash
   sudo useradd -m -u 3002 -g testers -s /bin/bash user2
   ```

5. Установите пароли для пользователей (в учебных целях используйте простой пароль `user123`):
   ```bash
   sudo passwd user1
   # Введите пароль: user123
   # Подтвердите пароль: user123
   
   sudo passwd user2
   # Введите пароль: user123
   # Подтвердите пароль: user123
   ```

6. Проверьте созданные учетные записи:
   ```bash
   id user1
   id user2
   grep "developers\|testers" /etc/group
   ```

> ✅ **Контрольная точка:** В выводе `id user1` должен отображаться UID=3001 и GID=2001 (developers). Для `user2` — UID=3002 и GID=2002 (testers).

### Задание 3: Работа с правами доступа ⭐ Базовое

1. Создайте тестовый файл от имени текущего пользователя:
   ```bash
   touch test_file.txt
   ```

2. Просмотрите текущие права доступа:
   ```bash
   ls -l test_file.txt
   ```

3. Установите права `0754` (владелец: полный доступ, группа: чтение+выполнение, остальные: чтение):
   ```bash
   chmod 0754 test_file.txt
   ```

4. Проверьте изменения:
   ```bash
   ls -l test_file.txt
   # Ожидаемый вывод: -rwxr-xr--
   ```

5. Создайте тестовый каталог:
   ```bash
   mkdir test_dir
   ```

6. Установите права `0755` на каталог:
   ```bash
   chmod 0755 test_dir
   ls -ld test_dir
   # Ожидаемый вывод: drwxr-xr-x
   ```

> ✅ **Контрольная точка:** После выполнения `chmod 0754 test_file.txt` вывод `ls -l` должен показывать права `-rwxr-xr--`.

> ✅ **Контрольная точка:** После всех изменений владелец файла должен быть `user2`, группа — `developers`, права — `0754`.

### Задание 5: Работа с маской umask ⭐ Базовое

> 💡 **Цель задания:** Понять, как `umask` влияет на права новых файлов и каталогов в восьмеричной системе.

1. Проверьте текущее значение `umask`:
   ```bash
   umask
   # Типичный вывод: 0022
   ```

2. Создайте файл и каталог при текущей маске:
   ```bash
   touch file_default.txt
   mkdir dir_default
   ```

3. Проверьте права созданных объектов:
   ```bash
   ls -l file_default.txt
   ls -ld dir_default
   # При umask 0022: файл 0644 (-rw-r--r--), каталог 0755 (drwxr-xr-x)
   ```

4. Измените `umask` на значение `0002` (расширенный доступ для группы):
   ```bash
   umask 0002
   umask
   # Вывод должен быть: 0002
   ```

5. Создайте новые объекты:
   ```bash
   touch file_umask0002.txt
   mkdir dir_umask0002
   ```

6. Проверьте права:
   ```bash
   ls -l file_umask0002.txt
   ls -ld dir_umask0002
   # При umask 0002: файл 0664 (-rw-rw-r--), каталог 0775 (drwxrwxr-x)
   ```

7. Измените `umask` на значение `0077` (максимальная изоляция):
   ```bash
   umask 0077
   umask
   # Вывод должен быть: 0077
   ```

8. Создайте новые объекты:
   ```bash
   touch file_umask0077.txt
   mkdir dir_umask0077
   ```

9. Проверьте права:
   ```bash
   ls -l file_umask0077.txt
   ls -ld dir_umask0077
   # При umask 0077: файл 0600 (-rw-------), каталог 0700 (drwx------)
   ```

10. Верните стандартное значение `umask`:
    ```bash
    umask 0022
    ```

> ✅ **Контрольная точка:** 
> - При `umask 0022` файл имеет права `0644`, каталог — `0755`
> - При `umask 0002` файл имеет права `0664`, каталог — `0775`
> - При `umask 0077` файл имеет права `0600`, каталог — `0700`
> 
> **Важно:** Все значения — в восьмеричной системе (обратите внимание на ведущие нули в выводе `ls -l` при использовании флага `-n`).

## Возможные ошибки и способы устранения

| Ошибка | Причина | Решение |
|--------|---------|---------|
| `useradd: user 'user1' already exists` | Пользователь уже существует в системе | Удалите существующего: `sudo userdel -r user1` |
| `useradd: cannot create directory /home/user1` | Нет прав или место на диске | Выполните от root, проверьте свободное место |
| `chmod: changing permissions of 'file': Operation not permitted` | Нет прав на изменение файла | Выполните от владельца файла или от root |
| `chown: invalid user: 'user1'` | Пользователь не существует | Проверьте имя: `id user1`, создайте пользователя |
| `passwd: Authentication token manipulation error` | Проблемы с `/etc/shadow` | Проверьте права: `ls -l /etc/shadow` (должны быть 0640, владелец root:shadow) |
| `groupadd: GID '2001' already exists` | GID уже занят другой группой | Выберите другой GID или удалите существующую группу |
| `умаск: команда не найдена` | Опечатка в команде | Используйте английское написание: `umask` |
| Неправильные права после `chmod` | Использование десятичной системы вместо восьмеричной | Всегда указывайте права в восьмеричном формате: `chmod 0755 file` |

### Полезные команды для диагностики
```bash
# Проверить текущую маску и рассчитать итоговые права (в восьмеричной системе)
umask && echo "Файл: 0$((8#666 - 8#$(umask))) | Каталог: 0$((8#777 - 8#$(umask)))"

# Просмотр прав в восьмеричном формате
ls -ln | awk '{print $1, $3, $4, $9}'

# Проверить, существует ли пользователь
id username 2>/dev/null && echo "Существует" || echo "Не существует"

# Найти все файлы с правами 0777 (потенциально небезопасные)
find /home -perm 0777 2>/dev/null
```

### Безопасность
- Никогда не устанавливайте одинаковые пароли для разных пользователей в production-среде
- Избегайте прав `0777` в рабочих системах — это нарушает принцип минимальных привилегий
- Для конфиденциальных данных используйте `umask 0077`
- Для совместной работы в команде используйте `umask 0002` или `0007`
- Регулярно проверяйте «забытые» учетные записи с помощью `getent passwd`
- **Всегда помните:** права доступа — это восьмеричные числа (`0755`), а не десятичные
