## Теоретическая часть

### 1. Архитектура пакетной системы ALT Linux

> 🔑 **Ключевое отличие от Debian/Ubuntu:** ALT Linux основан на **RPM**, а не на DEB. Это фундаментальное различие, которое необходимо усвоить.

| Характеристика | Debian/Ubuntu | ALT Linux |
|----------------|---------------|-----------|
| Формат пакетов | `.deb` | `.rpm` |
| Низкоуровневый менеджер | `dpkg` | `rpm` |
| Высокоуровневый менеджер | `apt` (классический) | `apt-rpm` (гибридный) |
| Формат репозиториев | `deb http://...` | `rpm [ветка] протокол://сервер путь компоненты` |

**Пакетные менеджеры в ALT Linux:**

| Менеджер | Назначение | Особенности |
|----------|------------|-------------|
| `rpm` | Низкоуровневая работа с пакетами | Требует ручного управления зависимостями |
| `apt-get` / `apt` | Высокоуровневый менеджер | Автоматически решает зависимости через `apt-rpm` |

```
Схема взаимодействия:
┌─────────────┐
│  Пользователь │
└──────┬──────┘
       │  (команды apt-get/apt)
       ▼
┌─────────────┐
│  apt-rpm    │ ← Автоматическое разрешение зависимостей
└──────┬──────┘
       │  (вызовы низкоуровневых операций)
       ▼
┌─────────────┐
│    rpm      │ ← Работа с .rpm пакетами
└──────┬──────┘
       ▼
┌─────────────┐
│ Файловая    │
│ система     │
└─────────────┘
```

### 2. Структура RPM-пакетов

- **Формат имени:** `имя-версия-релиз.архитектура.rpm`
- **Пример:** `nano-7.2-alt1.x86_64.rpm`
- **Содержимое пакета:**
  - Бинарные исполняемые файлы
  - Конфигурационные файлы (`/etc/`)
  - Скрипты установки/удаления
  - Метаданные о зависимостях

### 3. Репозитории ALT Linux

**Файлы конфигурации:** `/etc/apt/sources.list.d/*.list`

В современных версиях ALT Linux репозитории хранятся в отдельных файлах в директории `/etc/apt/sources.list.d/`. Каждая строка репозитория имеет формат:

```
rpm [ветка] протокол://сервер путь_к_ветке архитектура компоненты
```

**Примеры корректных строк для ветки p11:**

| Тип | Формат строки |
|-----|---------------|
| Официальный (ftp.altlinux.org) | `rpm [p11] http://ftp.altlinux.org/pub/distributions ALTLinux/p11/branch/x86_64 classic` |
| Зеркало Яндекса | `rpm [p11] http://mirror.yandex.ru/altlinux p11/branch/x86_64 classic` |

> ⚠️ **Важно:** Обратите внимание на **пробелы** в пути после домена! Это особенность синтаксиса `apt-rpm` в ALT Linux.

**Управление репозиториями:**
- Строка начинается с `#` → репозиторий **отключён**
- Строка без `#` в начале → репозиторий **активен**

**Типы репозиториев:**
- `classic` — стабильные пакеты основной ветки
- `gostcrypto` — пакеты с поддержкой ГОСТ-криптографии
- `updates` — обновления безопасности (в отдельных файлах)

### 4. Основные команды управления пакетами

| Команда | Назначение | Пример |
|---------|------------|--------|
| `apt-get update` | Обновление списка пакетов из репозиториев | `sudo apt-get update` |
| `apt-get upgrade` | Обновление установленных пакетов | `sudo apt-get upgrade` |
| `apt-get install` | Установка пакета с зависимостями | `sudo apt-get install nano` |
| `apt-get remove` | Удаление пакета (конфиги сохраняются) | `sudo apt-get remove nano` |
| `apt-get purge` | Полное удаление пакета с конфигами | `sudo apt-get purge nano` |
| `apt-get autoremove` | Удаление ненужных зависимостей | `sudo apt-get autoremove` |
| `apt-cache search` | Поиск пакетов по имени/описанию | `apt-cache search text editor` |
| `apt-cache show` | Просмотр информации о пакете | `apt-cache show nano` |
| `apt-cache depends` | Просмотр зависимостей пакета | `apt-cache depends nano` |
| `rpm -qa` | Список всех установленных RPM-пакетов | `rpm -qa \| head -20` |
| `rpm -qi` | Информация об установленном пакете | `rpm -qi nano` |
| `rpm -ql` | Список файлов, установленных пакетом | `rpm -ql nano` |
| `rpm -qf` | Определение пакета по файлу | `rpm -qf /bin/nano` |

> 💡 **Шпаргалка для студентов:**  
> `↑↓` — история команд | `Tab` — автодополнение | `Ctrl+C` — прервать выполнение | `#` — комментарий в конфигурационных файлах

## Практическая часть

> ✅ **Контрольная точка перед началом:** Убедитесь, что в системе есть интернет:
> ```bash
> ping -c 3 mirror.yandex.ru
> ```

### Задание 1: Управление репозиториями — отключение официального репозитория и активация зеркала Яндекса ⭐ Базовое

1. Просмотрите содержимое директории с файлами репозиториев:
   ```bash
   ls -la /etc/apt/sources.list.d/
   ```

2. Найдите файл с конфигурацией репозиториев (обычно `alt-p11.list` или `official-repositories.list`):
   ```bash
   cat /etc/apt/sources.list.d/*.list | grep -v "^#" | grep -v "^$"
   ```

3. Откройте файл репозиториев для редактирования (замените имя файла на актуальное):
   ```bash
   sudo nano /etc/apt/sources.list.d/alt-p11.list
   ```

4. **Отключите официальный репозиторий ALT** — добавьте символ `#` в начало строк с `ftp.altlinux.org`:
   ```text
   # rpm [p11] http://ftp.altlinux.org/pub/distributions ALTLinux/p11/branch/x86_64 classic
   # rpm [p11] http://ftp.altlinux.org/pub/distributions ALTLinux/p11/branch/x86_64-i586 classic
   # rpm [p11] http://ftp.altlinux.org/pub/distributions ALTLinux/p11/branch/noarch classic
   ```

5. **Активируйте зеркало Яндекса** — убедитесь, что строки с `mirror.yandex.ru` НЕ имеют символа `#` в начале:
   ```text
   rpm [p11] http://mirror.yandex.ru/altlinux p11/branch/x86_64 classic
   rpm [p11] http://mirror.yandex.ru/altlinux p11/branch/x86_64-i586 classic
   rpm [p11] http://mirror.yandex.ru/altlinux p11/branch/noarch classic
   ```

6. Сохраните файл и выйдите из редактора (`Ctrl+O`, `Enter`, `Ctrl+X` в nano).

7. Проверьте текущую активную конфигурацию репозиториев:
   ```bash
   grep -v "^#" /etc/apt/sources.list.d/*.list | grep -v "^$"
   ```

8. Обновите список доступных пакетов:
   ```bash
   sudo apt-get update
   ```

9. Убедитесь, что обновление прошло успешно и используется зеркало Яндекса:
   ```bash
   sudo apt-get update 2>&1 | grep -i yandex
   ```

> ✅ **Контрольная точка:** В выводе команды `apt-get update` должны присутствовать строки с `mirror.yandex.ru` и отсутствовать ошибки `404 Not Found`. Все строки с `ftp.altlinux.org` должны быть закомментированы (начинаться с `#`).

### Задание 2: Поиск и установка пакетов ⭐ Базовое

1. Найдите пакет для текстового редактора:
   ```bash
   apt-cache search editor | grep -i text
   ```

2. Просмотрите информацию о пакете `nano`:
   ```bash
   apt-cache show nano
   ```

3. Установите пакет:
   ```bash
   sudo apt-get install nano -y
   ```

4. Проверьте установку:
   ```bash
   which nano
   nano --version
   ```

> ✅ **Контрольная точка:** Команда `which nano` должна вернуть путь `/usr/bin/nano`.

### Задание 3: Обновление пакетов ⭐ Базовое

> 💡 **Типы обновлений:**
> - **Системные пакеты** — обновление всех установленных пакетов кроме ядра
> - **Ядро** — обновление только пакетов ядра (`kernel-image*`)
> - **Все пакеты** — полное обновление системы включая ядро
> - **Критические обновления** — обновления безопасности (в ALT Linux нет отдельной команды, используются все обновления из репозитория `updates`)

1. Обновите список пакетов:
   ```bash
   sudo apt-get update
   ```

2. Просмотрите список доступных обновлений:
   ```bash
   apt-get --dry-run upgrade
   ```

3. Выполните обновление всех пакетов:
   ```bash
   sudo apt-get upgrade -y
   ```

4. Для обновления только ядра (если требуется):
   ```bash
   # Сначала найдите пакеты ядра:
   apt-cache search kernel-image | grep p11
   
   # Установите конкретную версию ядра (пример):
   # sudo apt-get install kernel-image-std-def -y
   ```

> ✅ **Контрольная точка:** После обновления команда `apt-get upgrade` не должна предлагать новые обновления.

### Задание 4: Удаление пакетов ⭐ Базовое

> 💡 **Типы удаляемых пакетов:**
> - **Старые версии** — предыдущие версии пакетов (в ALT Linux хранятся в кэше `/var/cache/apt/archives/`)
> - **Неиспользуемые зависимости** — пакеты, установленные как зависимости, но больше не требуемые
> - **Временные/отладочные пакеты** — пакеты с суффиксом `-debug`, `-dbg`, `-devel`

1. Удалите пакет (оставив конфигурационные файлы):
   ```bash
   sudo apt-get remove nano -y
   ```

2. Проверьте, что пакет удалён:
   ```bash
   rpm -qa | grep nano
   which nano
   ```

3. Полностью удалите пакет с конфигурационными файлами:
   ```bash
   sudo apt-get purge nano -y
   ```

4. Удалите ненужные зависимости:
   ```bash
   sudo apt-get autoremove -y
   ```

5. Очистите кэш от старых версий пакетов:
   ```bash
   sudo apt-get clean
   ```

6. Найдите и удалите отладочные пакеты (если есть):
   ```bash
   # Поиск отладочных пакетов
   rpm -qa | grep -E "(debug|dbg|devel)$"
   
   # Удаление (пример):
   # sudo apt-get purge пакет-debug -y
   ```

> ✅ **Контрольная точка:** После `purge` и `autoremove` команда `rpm -qa | grep nano` не должна возвращать результатов. Кэш должен быть очищен: `du -sh /var/cache/apt/archives/` покажет минимальный размер.

### Задание 5: Работа с локальными RPM-пакетами ⭐⭐ Продвинутое

1. Создайте директорию для тестовых пакетов:
   ```bash
   mkdir -p ~/test_packages && cd ~/test_packages
   ```

2. Скачайте тестовый пакет из зеркала Яндекса:
   ```bash
   wget http://mirror.yandex.ru/altlinux/p11/branch/x86_64/RPMS.classic/nano-7.2-alt1.x86_64.rpm
   ```

3. Установите локальный пакет через `rpm`:
   ```bash
   sudo rpm -ivh nano-7.2-alt1.x86_64.rpm
   ```

4. Проверьте установку:
   ```bash
   rpm -q nano
   which nano
   ```

> ⚠️ **Важно:** При установке через `rpm` зависимости не разрешаются автоматически. Для автоматического разрешения используйте:
> ```bash
> sudo apt-get install ./nano-7.2-alt1.x86_64.rpm
> ```

### Задание 6: Диагностика и решение проблем ⭐⭐ Продвинутое

1. Сымитируйте проблему с зависимостями (принудительно удалите пакет без зависимостей):
   ```bash
   sudo rpm -e --nodeps nano 2>/dev/null || echo "Пакет nano не установлен — пропускаем шаг"
   ```

2. Проверьте состояние системы:
   ```bash
   sudo apt-get check
   ```

3. Исправьте сломанные зависимости:
   ```bash
   sudo apt-get --fix-broken install -y
   ```

4. Проверьте целостность системы:
   ```bash
   sudo apt-get install nano -y
   nano --version
   ```

> ✅ **Контрольная точка:** После `--fix-broken install` команда `apt-get check` не должна выводить ошибок.

## Возможные ошибки и способы устранения

| Ошибка | Причина | Решение |
|--------|---------|---------|
| `E: Unable to locate package` | Неверное имя пакета для ALT или не обновлён список пакетов | Проверьте имя через `apt-cache search`, выполните `apt-get update` |
| `E: Malformed line ... in source list` | Неправильный формат репозитория (неверные пробелы в пути) | Проверьте синтаксис: `rpm [ветка] протокол://сервер путь архитектура компоненты` |
| `error: Failed dependencies` | Установка через `rpm` без разрешения зависимостей | Используйте `apt-get install ./пакет.rpm` |
| `E: Could not get lock /var/lib/rpm/transaction` | Другой процесс использует `rpm` или `apt` | Дождитесь завершения или убейте процесс: `sudo killall apt apt-get rpm` |
| `GPG error: ... NO_PUBKEY` | Отсутствует ключ подписи репозитория | Выполните: `sudo apt-get install alt-gpg-keys` |
| `Недостаточно места в /var/cache/apt/archives/` | Мало свободного места на диске | Очистите кэш: `sudo apt-get clean` |

  ```bash
  sudo apt-get upgrade --dry-run
  ```
